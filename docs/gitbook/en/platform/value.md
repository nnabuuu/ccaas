# Core Value

## Every AI integration starts from zero. Yours shouldn't.

You explain your systems to the AI. You walk it through your rules, your data, your domain logic. It works. Then the session ends — and it forgets everything.

Next time, you explain it again.

The model isn't the problem. The platform is missing.

## The missing layer

KedgeAgentic is built on one idea: your business logic shouldn't live inside every prompt. It should live in the platform — persistent, consistent, available to every AI session automatically.

Two primitives make this possible:

| Layer | What It Does | Who Defines It |
|-------|-------------|----------------|
| **Skills** | Business logic — workflows, rules, domain knowledge | Solution developer |
| **MCP** | Tools & data — systems, APIs, information sources | Solution developer |
| **Platform** | Execution — Agent lifecycle, sessions, orchestration | KedgeAgentic |

Once defined, Skills and MCP tools are available in every Agent session. The AI knows your business from the first message.

## What this looks like in practice

**A monitoring team** defines what "normal" looks like in a Skill. The platform runs nightly checks using MCP-connected log systems and surfaces anomalies — no one on call required.

**A curriculum team** describes their lesson design standards in Skills. Teachers request a lesson plan; the AI produces one that matches their framework and pulls from the course library via MCP — from the first session, not the tenth.

**An analyst** defines a research framework in a Skill. Every analysis follows the structure, draws from real internal data via MCP, and produces consistent, repeatable outputs — whoever runs it.

## The result

AI that knows your business on day one. Not after weeks of prompt engineering. Not rebuilt every session. From the first interaction.
