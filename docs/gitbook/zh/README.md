# 即见Agentic 是什么

即见Agentic 是一个 **Agentic 服务平台** —— 你描述你的业务逻辑，平台将其作为生产级 AI 服务运行。

## 问题所在

AI 很强大，但它不懂你的业务。每次新的接入都要从头重建上下文：你的系统做什么、你的流程是什么、你的数据意味着什么。没有持久的业务上下文，AI 只能是一个通用助手，而不是领域专家。

## 解决方案

即见Agentic 引入两个核心原语，让 AI 永久理解你的业务：

- **Skills（技能）** —— 一次性描述你的业务逻辑。Skills 将工作流、规则和领域知识编码进来，AI 在每次会话中都能一致应用。
- **MCP** —— 一次性连接你的工具与数据。MCP（模型上下文协议）让 AI 能够结构化地访问你的系统、API 和信息来源。

平台处理其余一切：Agent Engine 生命周期、会话持久化、上下文管理、工具编排。你描述业务，平台负责运行。

## 核心理念

- **描述，而非编程** —— 用 Skills 表达业务逻辑，AI 负责执行
- **持久上下文** —— 业务知识跨会话积累，而不是每次重建
- **平台托管基础设施** —— Agent Engine、会话、工具均由平台管理
- **生产级可靠性** —— 面向真实工作负载构建：多租户隔离、审计日志、会话恢复

## 谁在使用即见Agentic

| 角色 | 构建什么 | 如何使用 |
|------|----------|----------|
| 开发者 | Solution（Skills + MCP + 前端） | 定义 Skills、配置 MCP、部署 |
| 业务团队 | 领域工作流 | 用 Skills 描述业务逻辑 |
| 终端用户 | Agentic 应用 | 与运行中的服务交互 |
