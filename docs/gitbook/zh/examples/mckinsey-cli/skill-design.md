# 纯 Skill 方案：零 MCP 架构

本页分析 McKinsey CLI 如何用单个精心设计的 Skill 完全替代 MCP 工具，以及这种模式何时适用于你自己的 Solution。

## 渐进式披露模式

### 问题：上下文窗口耗尽

朴素方案一次性加载所有指令：

```
SKILL.md (1130 行)
├── 方法论规则
├── 设计规范
├── 页面模板
├── 数据收集格式
├── 问题排查指南
└── 示例
```

1130 行的 Skill 在开始工作前就消耗大量上下文。加上 15-30 次网络搜索和页面生成，上下文耗尽将输出限制在约 15 页。

### 解决方案：导航核心 + 参考文件

V3.0 架构将 Skill 分为导航核心和按需参考文件：

```
SKILL.md (300 行)           ← 始终加载
├── 人设定义
├── 工作流概览（9 步）
├── 逐步导航
│   ├── STEP 1：直接执行
│   ├── STEP 2-3："加载 methodology.md" → 执行 → 释放
│   ├── STEP 4-5："加载 layouts.md, design-specs.md" → 执行 → 释放
│   ├── STEP 6-7："加载 excel-data-spec.md" → 执行 → 释放
│   ├── STEP 8："加载 delivery-summary.md"（如请求）
│   └── STEP 9："加载 troubleshooting.md"（如有问题）
└── 行为规则
```

每个 `加载 → 执行 → 释放` 循环使活跃上下文保持有界。

## 何时用 Skill vs. MCP

### 选择纯 Skill 当：

- **LLM 原生思考**：核心价值是分析、综合或创意生成
- **内置工具足够**：WebSearch 搜索数据、Write 生成文件、Read 读取参考
- **无外部集成**：不需要数据库、API 或认证服务
- **单一连贯工作流**：从输入到输出的逻辑流程
- **领域 = 指令**：业务逻辑可以用自然语言规则表达

### 选择 MCP 当：

- **外部数据源**：数据库、CRM、专有 API
- **专业计算**：LLM 无法可靠执行的算法
- **需要认证**：API 密钥、OAuth 令牌、服务凭证
- **多个独立工具**：不共享工作流的能力
- **实时数据**：两次调用间会变化的数据

### 混合方案

许多 Solution 结合两者：MCP 负责数据访问 + Skill 负责分析逻辑。智慧农服就是一个例子 —— MCP 工具获取结构化数据，而 Skill 控制分析工作流和输出格式。

## 增量生成：管理无界工作

### 问题

一份 25 页的麦肯锡 PPT 需要约 125 次网络搜索（每页 5 次）。同时将所有搜索结果加载到上下文中是不现实的。

### 解决方案：页面级上下文边界

```
┌─────────────────────────────┐
│  第 N 页上下文              │
│  ├── Dummy.md 页面规格      │
│  ├── 5 个搜索结果           │
│  ├── Excel 数据行           │  ← 单页上下文
│  └── PPTX 生成              │
├─────────────────────────────┤
│  清除上下文                 │  ← 用户确认后释放
├─────────────────────────────┤
│  第 N+1 页上下文            │
│  ├── Dummy.md 下一页        │
│  ├── 5 个新搜索结果         │  ← 全新上下文
│  └── ...                    │
└─────────────────────────────┘
```

关键机制：
- **用户确认门**：每页需要明确批准后才继续
- **上下文清理**：第 N 页的搜索结果不会带到第 N+1 页
- **页面依赖**：Skill 在生成依赖页面前检查前置页面是否存在

## 页面依赖管理

页面按依赖类型分类：

| 类型 | 标记 | 含义 | 示例 |
|------|-----|------|------|
| 独立 | `✅` | 可以任意顺序生成 | 封面页、独立分析页 |
| 前向依赖 | `⏩` | 依赖前面的页面 | 趋势汇总（需要数据页） |
| 后向依赖 | `⏪` | 最后生成 | 执行摘要（需要所有分析） |

Skill 分三轮生成页面：
1. **第一轮**：所有独立页面（任意顺序）
2. **第二轮**：前向依赖页面（前置条件完成后）
3. **第三轮**：后向依赖页面（执行摘要最后）

## 7 种麦肯锡页面模板

`layouts.md` 参考文件定义了标准化页面模板：

1. **标题 + 单图表** —— 单一数据故事（趋势图、饼图）
2. **标题 + 左右分栏** —— 对比布局（图表 + 解释）
3. **标题 + 2×2 矩阵** —— 战略定位（BCG 矩阵、机会图）
4. **标题 + 表格** —— 多维度对比
5. **标题 + 瀑布图** —— 增长分解（收入驱动因素）
6. **标题 + 时间轴** —— 历史演进
7. **洞察汇总** —— 章节总结（3-5 个关键洞察）

每个模板规定了精确的尺寸、定位和样式规则，确保输出质量一致。

## 自检协议

生成每页后，Skill 运行 6 项验证检查：

1. 布局匹配 Dummy 页面设计
2. 图表类型正确
3. 使用真实数据（非占位符）
4. 设计元素完整（颜色、字体、间距）
5. Excel 数据行已填充来源
6. 来源 URL 已记录以便追溯

如果任何检查失败，Skill 报告问题并等待用户指导。

## 可迁移模式

### 模式 1：导航核心 + 参考文件

对于超过 500 行指令的 Skill，拆分为：
- 核心 Skill（工作流导航，约 300 行）
- 参考文件（按需加载，使用后释放）

此模式适用于：报告生成器、代码生成器、课程构建器、设计系统。

### 模式 2：增量有界工作

当输出无界时（N 页、N 章、N 个分析），一次处理一个单元：
- 加载单元特定上下文
- 通过用户确认门执行
- 下一单元前清除上下文

### 模式 3：依赖感知生成

当输出单元存在依赖时，分类并排序：
- 独立单元优先（可并行）
- 前向依赖单元其次
- 后向依赖单元最后

### 模式 4：内置工具替代 MCP

在构建 MCP Server 之前，检查内置工具是否满足需求：
- `WebSearch` 替代外部搜索 API
- `Write` 替代文档生成服务
- `Read` 替代配置文件服务器
