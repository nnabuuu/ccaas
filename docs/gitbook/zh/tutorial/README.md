# Solution 构建教程

欢迎来到即见Agentic Solution 构建教程。这是一个完整的实战教程，将引导你从零开始构建一个 **Lesson Plan Designer（备课方案设计器）Solution** -- 从理解架构设计到部署上线。

## 你将构建什么

完成本教程后，你将拥有一个功能完整的备课方案设计器：

- 教师通过 Web 表单选择教材和章节
- AI Agent 协助生成教学目标、教学活动和评估方案
- AI 通过 `output_update` 协议将结构化数据直接写入表单
- 教师在保存前审核、编辑和批准 AI 生成的教案内容
- 教案可以持久化保存，支持跨会话恢复

```
┌──────────────────────────────────────────────┐
│        Lesson Plan Designer Solution         │
│                                              │
│  ┌─────────┐    ┌──────────┐    ┌─────────┐ │
│  │  前端    │◄──►│ Solution │◄──►│  CCAAS  │ │
│  │ (React)  │    │   后端   │    │   后端  │ │
│  └─────────┘    └──────────┘    └────┬────┘ │
│                                      │      │
│                                 ┌────▼────┐ │
│                                 │AI Agent │ │

│                                 │+ MCP    │ │
│                                 └─────────┘ │
└──────────────────────────────────────────────┘
```

## 前提条件

开始本教程前，你需要：

- 已安装 **Node.js 18+** 和 **npm**
- 具备 **TypeScript** 和 **React** 基础知识
- 了解 REST API 和 SSE 概念
- 已完成即见Agentic 安装（参见[安装与启动](../getting-started/installation.md)）

{% hint style="info" %}
**无需即见Agentic 使用经验。**本教程假设你从未使用过该平台。每个概念都会在深入代码之前给出清晰的解释。
{% endhint %}

## 如何使用本教程

### 推荐方式

1. **按顺序阅读。**每一章都建立在前一章的基础上。
2. **自己敲代码。**避免复制粘贴 -- 亲手输入有助于内化这些模式。
3. **运行检查点。**每个小节都以验证步骤结尾，检查点未通过前不要继续。
4. **完成练习。**每章都包含练习题，帮助你加深理解。

### 参考实现

本教程的规范参考实现是 **Lesson Plan Designer（备课方案设计器）**本身：

```
solutions/lesson-plan-designer/
```

这是一个生产级 Solution，展示了本教程中教授的所有模式。你可以用它来对比你的实现或在遇到困难时参考。但请先尝试自己构建。

{% hint style="warning" %}
**不要跳过设计章节**（第 1-3 章）。它们看起来可能比较抽象，但你在这些章节中学到的模式会直接影响第 6 章中的每个实现决策。跳过设计的开发者往往需要重写代码。
{% endhint %}

## 核心概念一览

以下是你将在本教程中学到的核心概念：

| 概念 | 说明 | 引入章节 |
|------|------|---------|
| **Solution** | 构建在即见Agentic 上的垂直应用 | 第 1 章 |
| **领域模型** | 代表业务实体的 TypeScript 接口 | 第 2 章 |
| **用户旅程** | 实现用户目标的一系列交互 | 第 3 章 |
| **中继架构** | CCAAS 作为前端和 AI Agent 之间的中继 | 第 4 章 |
| **output\_update** | 将 AI 输出同步到前端表单的协议 | 第 5 章 |
| **SyncField** | 可以接收 AI 生成数据的表单字段 | 第 5 章 |
| **write\_output** | AI 调用来写入结构化数据的 MCP 工具 | 第 5 章 |
| **Skill** | 定义 AI Agent 行为的 Markdown 文件 | 第 6.4 章 |
| **MCP Server** | 提供 AI Agent 可调用工具的服务 | 第 6.3 章 |
