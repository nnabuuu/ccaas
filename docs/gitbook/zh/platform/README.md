# 平台概述

即见Agentic 是一个 **Agentic 服务平台**，将业务描述转化为生产级 AI 服务。开发者定义 Skills、连接 MCP 工具，平台管理其余一切。

## 平台定位

即见Agentic **不是**一个 AI 模型或模型封装器，而是一个 **Skills + MCP 执行平台**，提供：

- **Skill 路由** —— 用 Skills 描述的业务逻辑，由平台智能匹配并交由合适的 Agent 执行
- **MCP 工具编排** —— 通过 MCP 连接的工具和数据源，无需每次会话重新配置即可供所有 Agent 使用
- **会话管理** —— Agent Engine 生命周期、会话持久化、上下文连续性均由平台处理
- **多租户基础设施** —— 为每个租户和 Solution 提供隔离的生产级执行环境

## 平台组成

即见Agentic 平台由以下核心组件构成：

| 组件 | 职责 |
|------|------|
| **Agent Engine** | AI Agent 执行引擎，管理会话生命周期 |
| **Skill System** | 技能路由与管理，支持关键词/模式/意图匹配 |
| **MCP Server** | 工具服务层，提供 AI Agent 可调用的外部工具 |
| **Solution Framework** | 面向垂直场景的完整应用框架 |
| **Vue SDK** | 前端集成 SDK，提供响应式状态管理 |
| **Admin Console** | 管理后台，提供租户管理、分析仪表盘 |

## 技术优势

1. **会话管理** —— 完整的会话持久化与上下文恢复，每次对话都建立在之前交互的基础上
2. **Skill 路由** —— 基于触发器的智能路由，支持关键词、正则表达式、意图、上下文匹配
3. **MCP 集成** —— 标准化的工具协议，支持 REST API 适配和自定义工具开发
4. **实时流式传输** —— 基于 SSE 的事件流，支持思考过程、工具使用、输出生成的实时展示
5. **多租户架构** —— 租户级别的资源隔离与配额管理
6. **消息持久化** —— 完整的对话历史存储，支持会话恢复
