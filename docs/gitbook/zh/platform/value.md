# 核心价值

## 每次 AI 接入都从零开始。没必要这样。

你向 AI 解释你的系统，告诉它你的规则、数据、业务逻辑。它跑通了。然后会话结束——它把一切都忘了。

下次，你重新解释一遍。

问题不在模型，在于缺少一个平台。

## 缺失的那一层

即见Agentic 建立在一个简单的想法上：你的业务逻辑不应该活在每条 prompt 里，而应该活在平台中——持久存在，跨所有 AI 会话一致可用，不需要每次重新教。

两个核心原语让这成为可能：

**Skills** —— 一次描述你的业务逻辑。工作流、规则、领域知识。AI 在每次会话中都会应用，不需要反复告知。

**MCP** —— 一次连接你的工具与数据。AI 直接操作你系统中的真实信息，而不是你对它的描述。

平台处理其余一切：Agent 生命周期、会话连续性、上下文管理、工具编排。你描述业务，它运行。

## 实际场景

**运维团队**在 Skill 里定义"正常"的标准。平台每晚通过 MCP 连接的日志系统自动巡检，标记异常——不需要人值守。

**课程团队**在 Skills 里描述教案设计规范。教师发出请求，AI 产出符合框架、通过 MCP 连接课程库的教案——从第一次会话起，不是第十次磨合之后。

**分析师**在 Skill 里定义研究框架。每次分析都遵循这个结构，从 MCP 连接的内部数据取数，输出一致、可复现的结果——无论谁来运行。

## 结果

AI 从第一天起就懂你的业务。不需要数周的 prompt 调试，不需要每次会话重建，从第一次交互开始。
