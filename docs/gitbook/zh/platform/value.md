# 核心价值：Human-in-the-Loop Co-Pilot

## 为什么需要 Human-in-the-Loop

企业在采用 AI 时面临两难选择：

| 方案 | 问题 |
|------|------|
| **全自动化** | 风险高、不可控、缺乏审计追踪、出错成本大 |
| **纯人工** | 效率低、成本高、无法规模化 |
| **LoopAI 方案** | AI 执行 + 人类监督，兼顾效率与安全 |

LoopAI 采用 **Co-Pilot 模式**：AI Agent 负责执行重复性任务，人类操作员负责审核、修正、决策。每一步操作都可追溯，每次修改都有版本记录。

## 核心价值主张

### 1. 协作，而非替代

LoopAI 不追求完全替代人类，而是让 AI 成为人类的高效助手：

- AI 产出初稿，人类审核修改
- 人类修改后，AI 理解变更并继续推进（Continue 工作流）
- 全程版本控制，支持一键回滚

### 2. AI 引擎无关

LoopAI 不绑定特定 AI 供应商，通过标准化适配器接口支持多种 AI 后端：

- 商业 Agent 运行时（如 Claude Code、Cursor、Windsurf）
- 开源 Agent（如 OpenCode、Aider、Continue）
- 原生 LLM API（如 OpenAI、Anthropic、Gemini）
- 企业自研 Agent

### 3. 遗留系统无缝集成

70% 的企业应用已运行 10 年以上，缺乏现代 API。LoopAI 提供多种集成方式：

- **浏览器自动化** —— 通过 Playwright 操作 Web 界面
- **REST API 集成** —— 标准化的 API 连接器
- **Webhook** —— 事件驱动的系统通知
- **预览模式** —— 提交前人工确认，附带截图审计

### 4. 企业级治理

- **多租户隔离** —— 数据库级别的租户隔离
- **审计日志** —— 不可变的完整操作日志
- **版本控制** —— 每次变更自动创建版本，支持差异对比和一键恢复
- **权限管理** —— 基于角色的访问控制（RBAC）
- **合规支持** —— AES-256 静态加密 + TLS 1.3 传输加密

## 典型应用场景

### 财务与会计

- 发票自动识别与录入，人类审核确认
- 费用报告自动生成，财务人员审批
- 月度/季度报表 AI 初稿，分析师修改定稿

### 运营与管理

- 数据批量录入，运营人员抽查验证
- 系统间数据迁移，管理员监督执行
- 文档自动整理归档，人工复核分类

### 教育与培训

- AI 辅助教案设计，教师审核修改
- 题目智能解析与讲解，教研员优化内容
- 课程资源自动匹配，教师确认选用
