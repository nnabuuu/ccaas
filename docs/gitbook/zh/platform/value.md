# 核心价值：描述你的业务，AI 完成其余

## 问题：AI 不懂你的业务

没有持久上下文，每次 AI 交互都从零开始。你解释你的系统、你的流程、你的领域规则——下次会话，再来一遍。AI 只能是通用助手，而不是领域专家。

这不是模型质量问题，而是平台问题。

## 解法：Skills + MCP

即见Agentic 通过两个核心原语解决这个问题：

| 层级 | 作用 | 由谁定义 |
|------|------|----------|
| **Skills** | 业务逻辑 —— 工作流、规则、领域知识 | Solution 开发者 |
| **MCP** | 工具与数据 —— 系统、API、信息来源 | Solution 开发者 |
| **平台** | 执行 —— Agent 生命周期、会话、编排 | 即见Agentic |

一旦定义，Skills 和 MCP 工具在每次 Agent 会话中都可用。AI 从第一条消息起就懂你的业务。

## 核心价值主张

### 1. 一次描述，处处应用

Skills 永久编码你的业务逻辑：

- 在 Skill 中定义工作流，AI 在所有会话中一致应用
- 更新 Skill，所有后续会话立即使用新逻辑
- 无需逐会话提示，无需重建上下文

### 2. 通过 MCP 连接工具与数据

MCP 让 AI 结构化地访问你的系统：

- 将 REST API、数据库、服务作为 MCP 工具接入
- AI 可以查询、读取并基于真实数据行动，而不只是谈论它
- 工具无需每次会话重新配置，所有 Agent 均可使用

### 3. 平台托管基础设施

平台处理你不需要操心的事：

- **Agent Engine 生命周期** —— 自动启动、停止、恢复 Agent 会话
- **会话持久化** —— 对话历史和上下文跨会话保留
- **上下文管理** —— Skills 和 MCP 的业务上下文流入每次交互
- **工具编排** —— MCP 工具自动路由、调用，结果自动集成

### 4. 默认企业级

- **多租户隔离** —— 数据库级别的租户隔离
- **审计日志** —— 不可变的完整操作日志
- **会话恢复** —— 中断的会话可恢复，上下文不丢失
- **权限管理** —— 基于角色的访问控制（RBAC）

## 典型应用场景

### 定时执行

- 夜间系统健康检查，AI 生成巡检报告
- 定时数据分析与异常检测
- 自动化报告，附带领域专属解读

### 教案设计

- AI 使用编码了课程标准的 Skills 设计教案
- 教师审核修改，AI 在上下文中应用编辑
- MCP 连接课程库、测评工具和学习管理系统

### 商业分析

- AI 使用 Skills 中定义的麦肯锡式框架进行结构化分析
- 通过 MCP 连接内部数据源获取真实数据
- 输出一致、可复现的分析结果

### 系统巡检

- AI 使用 Skills（定义了你的基础设施"正常"是什么）监控系统健康
- MCP 工具连接日志、指标和告警系统
- 异常触发结构化的排查工作流
