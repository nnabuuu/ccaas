# 平台架构

## 架构总览

即见Agentic 采用分层架构设计，每一层职责明确，可独立扩展。

```
┌─────────────────────────────────────────────┐
│              前端应用层                        │
│   Solution 前端 · 管理后台                     │
├─────────────────────────────────────────────┤
│              API 网关层                       │
│   认证鉴权 · 会话管理 · 消息路由 · 限流         │
├─────────────────────────────────────────────┤
│              编排层                           │
│   Agent Engine · Skill Router · 租户注册      │
├─────────────────────────────────────────────┤
│              执行层                           │
│   AI Agent 进程 · 工具调用 · 定时任务 · 文件管理  │
├─────────────────────────────────────────────┤
│              MCP 服务层                       │
│   内置工具 · REST 适配器 · 自定义 MCP Server   │
├─────────────────────────────────────────────┤
│              持久化层                         │
│   消息存储 · 版本管理 · 文件存储 · 审计日志      │
└─────────────────────────────────────────────┘
```

## 核心架构组件

### Agent Engine

{% hint style="info" %}
**平台内部组件**：AgentEngine 由即见Agentic 平台统一管理。作为 Solution 开发者或平台用户，你通过 API 和 SDK 使用 AI 能力，无需配置或管理底层引擎。
{% endhint %}

Agent Engine 是平台的核心执行引擎，负责：

- **会话生命周期管理** —— 创建、恢复、终止 AI Agent 会话
- **进程隔离** —— 每个会话独立运行，互不干扰
- **事件流式传输** —— 实时推送 AI 思考过程、工具使用、输出结果
- **上下文管理** —— 维护对话历史与工作空间状态

### Skill Router

Skill Router 实现智能任务分发：

- **触发器匹配** —— 基于关键词、正则表达式、意图、上下文的多维度匹配
- **优先级管理** —— 支持触发器优先级排序
- **版本控制** —— Skill 支持版本管理与发布流程

### MCP 服务层

MCP（Model Context Protocol）服务层提供标准化的工具接口：

- **REST 适配器** —— 将 REST API 封装为 MCP 工具
- **工具池管理** —— 管理工具服务的生命周期与健康检查
- **多种认证方式** —— 支持 OAuth2、API Key、Bearer Token、Basic Auth

### 调度引擎

调度引擎实现自动化的无人值守任务执行：

- **动态调度注册** —— 通过 `SchedulerRegistry` 在运行时注册和管理 cron 作业、间隔任务和延时任务
- **无头执行** —— 无需 WebSocket 即可启动 AgentEngine，在进程内收集结果
- **并发与重试** —— 每任务并发限制和可配置的重试策略
- **漏跑恢复** —— 启动时检测并触发错过调度时间的任务

### Solution Framework

Solution 是面向垂直场景的完整应用框架：

```
Solution
├── frontend/     # 前端应用（React/Vue）
├── backend/      # 业务后端
├── mcp-server/   # 工具服务
├── skills/       # AI Skill 定义
└── solution.json # 配置文件
```

每个 Solution 可以：
- 定义专属的 MCP 工具
- 编写定制化的 Skill
- 构建独立的前端界面
- 接入平台的全部基础能力
- **（可选）** 用 Ontology 描述领域类型 + 用 TriggerDef 声明工作流，让平台 WorkflowEngine 自动调度。详见 [Ontology & Workflow](../ontology/README.md) 章节。

## 数据流

一次完整的用户交互遵循以下数据流：

```
用户输入
  │
  ▼
API 网关 ── 认证 ── 租户解析
  │
  ▼
Skill Router ── 触发器匹配 ── 选择 Skill
  │
  ▼
Agent Engine ── 创建/恢复会话 ── 注入上下文
  │
  ▼
AI Agent 进程
  │── 思考过程 ──→ 实时推送到前端
  │── 工具调用 ──→ MCP Server 执行
  │── 输出结果 ──→ 结构化数据同步
  │
  ▼
前端应用 ── 渲染结果 ── 用户交互
```

## 部署架构

即见Agentic 支持灵活的部署方式：

| 部署模式 | 适用场景 | 说明 |
|----------|----------|------|
| **本地开发** | 开发调试 | 单机运行，SQLite 存储 |
| **单机部署** | 小团队 | Docker Compose 一键部署 |
| **集群部署** | 企业生产 | Kubernetes 编排，PostgreSQL 存储 |
| **私有化部署** | 安全敏感 | 完全隔离的企业内网部署 |
