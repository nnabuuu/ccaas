# 核心能力

## AI 任务执行

即见Agentic Agent Engine 提供自然语言驱动的任务执行能力：

- **自然语言输入** —— 用户用自然语言描述任务需求，支持附件上传
- **实时流式传输** —— AI 思考过程、工具使用、输出结果实时推送到前端
- **可视化执行状态** —— 实时展示 AI 当前动作（思考、搜索、执行、生成）
- **任务取消** —— 随时中断正在执行的任务
- **交互式提示** —— Agent 可在执行中暂停，向用户提出结构化问题；用户通过 UI 卡片或自定义向导回答
- **会话恢复** —— 断线重连后自动恢复会话状态

## Skill 系统

Skill 是即见Agentic 的核心抽象，定义了 AI Agent 在特定场景下的行为：

### Skill 类型

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| **Prompt** | 指令型 Skill，定义角色和行为规范 | 大多数场景 |
| **Workflow** | 多步骤工作流 Skill | 复杂流程 |
| **Sub-agent** | 子 Agent Skill，可配置独立模型 | 专业子任务 |
| **Tool-config** | 工具配置型 Skill | 工具编排 |

### 智能路由

Skill Router 支持四种触发方式，自动将用户请求分发到最合适的 Skill：

- **关键词匹配** —— 消息包含指定关键词时触发
- **正则表达式** —— 消息匹配正则模式时触发
- **意图识别** —— 基于语义理解的意图匹配
- **上下文匹配** —— 基于当前页面、表单状态等上下文触发

## MCP 工具集成

MCP（Model Context Protocol）提供标准化的工具接口，让 AI Agent 可以调用外部服务：

### 内置工具

- **write\_output** —— 结构化输出工具，将 AI 生成的内容同步到前端表单
- **文件操作** —— 读写工作空间内的文件

### REST 适配器

将现有的 REST API 封装为 MCP 工具，无需修改现有服务：

- 支持 GET/POST/PUT/DELETE 方法
- 灵活的参数映射
- 多种认证方式（OAuth2、API Key、Bearer Token、Basic Auth）
- 内置健康检查与重试机制

### 自定义工具开发

开发者可以创建自定义 MCP Server，提供领域特定的工具。

## 实时协作

### 结构化输出同步

AI Agent 通过 `write_output` 工具生成结构化数据，实时同步到前端表单：

- **增量更新** —— 支持 set、append、merge 三种操作
- **进度追踪** —— 实时展示生成进度
- **字段级同步** —— 精确到表单字段级别的数据同步

### SSE 事件流

平台通过 SSE 推送丰富的实时事件：

| 事件 | 说明 |
|------|------|
| `text_delta` | AI 文本流式输出 |
| `output_update` | 结构化数据更新 |
| `agent_status` | Agent 状态变化（空闲、思考、执行、完成） |
| `tool_activity` | 工具使用活动（开始、进行、结束）；也承载 AskUserQuestion 的交互式提示 payload |
| `todo_update` | 任务列表更新 |
| `token_usage` | Token 使用统计 |

## 定时任务执行

即见Agentic 支持后台定时任务调度，实现无人值守的自动化 AI 操作：

- **Cron 调度** —— 按 Cron 表达式定时执行（如每天凌晨 4 点）
- **间隔调度** —— 按固定间隔重复执行（如每 60 秒）
- **一次性调度** —— 在指定日期时间执行一次
- **后台执行** —— 无需实时连接，后台自动运行
- **并发控制** —— 可配置每个任务的最大并发执行数
- **重试逻辑** —— 失败时自动重试，支持可配置的重试延迟
- **执行历史** —— 完整的执行审计记录，包含结果、Token 用量和耗时
- **实时通知** —— 通过 SSE 向 `scheduler:{solutionId}` 房间推送事件
- **漏跑检测** —— 服务重启时自动检测并触发错过调度的任务

### 使用场景

| 场景 | 调度类型 | 示例 |
|------|---------|------|
| 每日内容聚合 | Cron | `0 4 * * *`（每天凌晨 4 点） |
| 定期监控 | 间隔 | 每 5 分钟 |
| 一次性报告生成 | 一次性 | 指定日期时间 |
| 自动邮件摘要 | Cron | `0 9 * * 1`（周一上午 9 点） |

## Context Layer

@ 引用系统，让聊天用户以内联方式提及业务实体。

- **装饰器驱动** —— 控制器上使用 `@Referenceable` + 服务方法上使用 `@Tracked`
- **7 个 REST 端点** —— 实体类型、推荐、浏览、搜索、解析、活动记录、快捷方式
- **Tool-based 架构** —— Agent 按需通过 MCP tools 获取实体数据，而非预注入
- **Redis 推荐引擎** —— 活动追踪，5 个内置动作 + 通过 `ActivityActionConfig` 扩展自定义动作
- **关系树** —— 从 ORM `@ManyToOne` 元数据自动推断；支持钻入导航
- **前端** —— `<AtPicker />` 组件，支持最近使用、搜索、钻入、面包屑和键盘导航

## 多租户管理

- **租户隔离** —— 每个租户拥有独立的配置、Skill、数据
- **API Key 认证** —— 基于 Scope 的细粒度权限控制
- **配额管理** —— 最大会话数、每日 Token 限额、可用模型限制
- **用量分析** —— Token 使用统计、会话分析、成本追踪

## 消息持久化

- **完整对话历史** —— 所有用户消息和 AI 回复均持久化存储
- **会话恢复** —— 支持会话断线重连和历史恢复
- **文件关联** —— AI 生成的文件自动关联到会话
- **导出支持** —— 支持对话历史导出
