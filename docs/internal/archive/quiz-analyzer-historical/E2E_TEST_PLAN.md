# Quiz Analyzer - End-to-End Test Plan

## 测试概述

验证三栏布局的完整分析流程，确保从用户输入到结果展示的所有环节正常工作。

---

## 测试环境

### 服务启动检查清单

- [ ] CCAAS Backend (http://localhost:3001)
- [ ] Quiz Backend (http://localhost:3005)
- [ ] MCP Server (stdio mode, auto-start)
- [ ] Quiz Frontend (http://localhost:5282)

### 启动命令

```bash
# Terminal 1: CCAAS Backend
cd packages/backend
npm run dev

# Terminal 2: Quiz Backend
cd solutions/quiz-analyzer/backend
npm run start:dev

# Terminal 3: Quiz Frontend
cd solutions/quiz-analyzer/frontend
npm run dev
```

## 测试场景

### 场景 1: 纯题目分析（选择题）✅

**测试目标**: 验证基本分析流程

**输入数据**:
```
题目内容:
已知函数 f(x) = x² - 2x + 1，求 f(x) 的最小值。
A. -1
B. 0
C. 1
D. 2

参考答案:
B

学生答案:
(留空)
```

**预期结果**:

| 时间点 | 中栏展示 | 右栏展示 |
|--------|----------|----------|
| t=0s | 空状态提示 | 等待输入 |
| t=1s | "解析中..." | 加载动画 |
| t=2s | ✅ 题干: "已知函数..." | AI: "题目解析完成" |
| t=2s | ✅ 选项: A/B/C/D | |
| t=2s | ✅ 正确答案: B (绿色) | |
| t=2s | ✅ 题型: 选择题 | |
| t=4s | ✅ 知识点: "二次函数", "函数最值" | AI: "已标注 2 个知识点" |
| t=5s | ✅ 目录: 九年级 > 函数 > 二次函数 | AI: "已定位到目录" |
| t=6s | ✅ 难度: 3/5 (橙色方块) | AI: "难度等级: 3/5" |
| t=8s | (不变) | ✅ 解题思路 (Markdown) |

**验证检查点**:
- [ ] 题干正确提取，无截断
- [ ] 选项 A/B/C/D 正确分离
- [ ] 正确答案 B 高亮显示
- [ ] 知识点标签有置信度星标 (>0.8)
- [ ] 目录面包屑可点击/展开
- [ ] 难度条颜色正确 (橙色已达到，灰色未达到)
- [ ] 解题思路 Markdown 正确渲染

**性能要求**:
- [ ] 总分析时间 < 10 秒
- [ ] 每步更新延迟 < 2 秒
- [ ] 无卡顿或闪烁

### 场景 3: 填空题分析 📝

**测试目标**: 验证不同题型的解析能力

**输入数据**:
```
题目内容:
若 x² - 5x + 6 = 0，则 x = ____

参考答案:
2 或 3

学生答案:
(留空)
```

**预期结果**:

| 检查点 | 预期 |
|--------|------|
| 题型 | ✅ "填空题" |
| 选项 | ✅ 空数组 (无选项) |
| 正确答案 | ✅ "2 或 3" |
| 知识点 | ✅ "一元二次方程", "因式分解" |

**验证检查点**:
- [ ] 题型正确识别为 "填空题"
- [ ] 不显示选项区域
- [ ] 答案格式正确显示
- [ ] 知识点标注准确

### 场景 4: 多知识点综合题 🔗

**测试目标**: 验证多知识点标注能力

**输入数据**:
```
题目内容:
在平面直角坐标系中，抛物线 y = ax² + bx + c 经过点 A(1, 0)、B(3, 0) 和 C(0, -3)。
(1) 求抛物线的解析式
(2) 求抛物线的顶点坐标

参考答案:
(1) y = x² - 4x + 3
(2) (2, -1)

学生答案:
(留空)
```

**预期结果**:

| 检查点 | 预期 |
|--------|------|
| 知识点数量 | ✅ ≥ 3 个 |
| 知识点示例 | ✅ "二次函数", "待定系数法", "顶点坐标" |
| 难度 | ✅ ≥ 4/5 (综合题) |

**验证检查点**:
- [ ] 至少标注 3 个知识点
- [ ] 包含 "二次函数" 和 "待定系数法"
- [ ] 难度评估合理 (≥4)
- [ ] 解题思路分步骤展示

### 场景 5: 对话继续分析 💬

**测试目标**: 验证上下文保持和后续交互

**步骤**:
1. 完成场景 1 的题目分析
2. 在聊天框输入: "详细解释一下二次函数的配方法"
3. 观察 AI 回答

**预期结果**:
- [ ] AI 基于当前题目上下文回答
- [ ] 回答包含具体示例 (f(x) = x² - 2x + 1)
- [ ] 不需要重新分析题目
- [ ] 聊天历史完整保留

---

### 场景 6: 新对话重置 🔄

**测试目标**: 验证清空对话功能

**步骤**:
1. 完成场景 1 的题目分析
2. 点击 "新对话" 按钮
3. 观察界面状态

**预期结果**:
- [ ] 左栏输入表单保持不变 (不清空)
- [ ] 中栏清空，回到空状态
- [ ] 右栏聊天历史清空
- [ ] 会话 ID 改变
- [ ] 可以重新开始新的分析

---

### 场景 7: 网络断开恢复 🔌

**测试目标**: 验证网络异常处理

**步骤**:
1. 开始分析题目
2. 在分析过程中（t=3s）停止 CCAAS Backend
3. 观察前端反应
4. 重启 CCAAS Backend
5. 观察恢复情况

**预期结果**:
- [ ] 显示 "连接断开" 提示
- [ ] 分析暂停，不崩溃
- [ ] 后端恢复后自动重连
- [ ] 可以继续新的分析
- [ ] 断开前的结果保留在中栏

---

### 场景 8: 工具调用失败 ⚠️

**测试目标**: 验证 MCP 工具失败的容错性

**模拟方式**:
- 停止 MCP Server (手动 kill process)
- 或修改 MCP 工具返回错误

**预期结果**:
- [ ] 显示友好错误提示
- [ ] 不中断整个流程
- [ ] 使用默认值/跳过失败步骤
- [ ] 聊天框显示具体错误原因
