## V1 Generator 完成

### 本轮修改摘要

**核心问题**：历史上所有轮次（得分 27-31）的主要失败原因都是 **HTTP 429 rate limiting**，导致 83-100% 的题目分析完全失败。

**V1 策略：减少 token 消耗，提高完成率**

| 修改 | 目标 |
|------|------|
| 新增"静默高效模式"（最高优先级规则） | 禁止 agent 输出分析文字，每步只输出 `✅ N`，减少 50%+ token 消耗 |
| HTTP 429 专项容错策略 | 遇到限流不放弃，记住失败字段，到补漏扫描时再重试 |
| 极端情况降级策略 | >3 个核心字段失败时放弃非核心步骤，集中补输出核心字段 |
| 跳过 `generate_thinking_process_template` | 每题节省一次工具调用 |
| 强化答案验证（静默验证） | 验证过程不输出到聊天，节省 token 同时确保正确性 |
| 精简 prompt 自身长度 | 缩减实例、持久化 section，减少 prompt token |

**修改量**：约 50 行变更 / 578 行总量 = ~8.7%（远在 30% 限制内）

**预期分数**：27-31 → 60-75（主要取决于 429 问题是否因 token 节省而缓解）
