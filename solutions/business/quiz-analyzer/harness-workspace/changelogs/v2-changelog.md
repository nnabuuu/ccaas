# v2 Changelog

## 修改摘要

强化重试禁令（🔴绝对禁止规则）、增加选项格式反面示例、要求 formula 字段、增加 KP 覆盖规则——针对 v1 (80分) 扣分最多的 4 个问题逐一修复。

## 修改详情

- [行 57-62] **工具调用纪律 §2 重写**：将原来的一句话"失败不重试"替换为 🔴 标记的"绝对禁止重试规则"，列出具体错误类型（Sibling tool call errored、HTTP 429、超时），明确"每个字段最多调用 1 次 write_output"，并说明违反后果（token 浪费导致核心字段缺失）。
  - **原因**：v1 中 quizAnalysis 被重试 ~20 次，直接导致 analysisStrategy 和 knowledgeGapAnalysis 两个核心字段未输出（D1 -5分），且触发重复写入 penalty（-6分）。

- [行 203-210] **步骤1 parsedContent options 规则增强**：增加 🔴 标记的"选项格式对比"区块，用 ❌/✅ 对比展示错误和正确的选项格式，并补充说明"去掉所有 A./B./C./D. 前缀及其后面的空格"。
  - **原因**：v1 中 options 保留了"A. 两个不相等的实数根"格式，D3 被扣 3 加权分。

- [行 274-275] **步骤3 quiz_analysis 失败处理强化**：将原来的"不要重试，直接进入步骤4"替换为更醒目的 🔴 标记说明，并明确 quiz_analysis 不是核心字段，失败不影响评分。
  - **原因**：v1 agent 在 quiz_analysis 上浪费大量 token 重试，是核心字段缺失的直接原因。

- [行 371-383] **步骤6 solutionSteps formula 字段要求**：将原来的"formula 和 reasoning 为可选"改为"涉及数学公式/计算的步骤必须提供 formula 字段"，增加包含 formula 的步骤示例 JSON。
  - **原因**：v1 所有步骤均无 formula 字段，公式嵌在 description 中，D4 被扣 3 加权分。

- [行 239-245] **步骤2 知识点覆盖规则**：在置信度规则之前新增"知识点覆盖规则"区块，要求同时标注具体叶节点和所属大类作为独立标签（2-3个标签），并给出示例。
  - **原因**：v1 只标注了"一元二次方程根的判别式"一个叶节点，缺少"一元二次方程"的独立标签，D5 被扣 3 加权分。

## 对应维度

- D1 (Field Completion): 🔴 强化重试禁令，确保 token 预算留给所有核心字段，避免 quizAnalysis 重试导致后续字段被截断
- D2 (Answer Correctness): 无修改（v1 已满分 5/5）
- D3 (Quiz Type): 增加选项格式反面示例 ❌/✅ 对比，明确去除前缀规则
- D4 (Solution Steps): 将 formula 从"可选"改为"涉及计算时必须提供"，增加示例
- D5 (KP Tags): 增加"同时标注具体知识点和所属大类"规则，建议 2-3 个标签
- D6 (Geometry): 无修改（v1 已满分 5/5）
