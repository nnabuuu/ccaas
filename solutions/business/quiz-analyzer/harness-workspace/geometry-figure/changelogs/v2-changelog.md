# v2 Changelog

## 修改摘要
扩展坐标配方覆盖矩形/菱形/圆/椭圆、增加标准配色表、强化"必须输出"防护、补充顺时针旋转公式和触发条件。

## 背景

v1 评分 30/100，但**根因是基础设施问题**（HTTP 429 + benchmarkSize=2），不是 prompt 质量。v1 的 prompt 改进从未被实际测试。v2 策略：在 v1 基础上补强最可能导致失败的薄弱环节——缺少特定题型配方、缺少输出保障、顺时针旋转无公式。

## 修改详情

### geometry-problem-figure/SKILL.md

- **[行 28-43] 扩展坐标配方表**：新增矩形（D=(0,0),C=(w,0),B=(w,h),A=(0,h)）、菱形（中心在原点，对角线沿坐标轴）、圆外点（O=(0,0),P=(d,0)）、椭圆（O=(0,0),F1/F2=(±c,0)）四种配方。覆盖 benchmark 中 geo-004(矩形)、geo-006(圆)、geo-007(椭圆)、geo-008(菱形) 的坐标布局需求。（D3, D4）

- **[行 165-173] 新增标准配色参考**：定义 7 种标准颜色（主色/红/橙/绿/紫/灰/中心），减少 LLM 随意选色导致的视觉不一致。（D5）

- **[行 207-210] 新增"第零步：确保输出完整 JSON"**：明确要求即使对题型不确定也必须输出包含 kind/bbox/elements 的完整 JSON。空输出=自动0分。这是防止 LLM 在遇到不熟悉题型时选择不输出的防护。（D1）

- **[行 225-227] 扩展 self-check**：新增 ellipse fillColor:none 检查、ellipse 半长轴参数检查、解析几何题坐标轴辅助线检查。（D3, D5）

- **[行 235-236] 扩展常见错误**：新增 ellipse parents 错误（2a vs a）和矩形/菱形折叠题忘记读 reference file 的警告。（D3, D7）

### geometry-solution-figure/SKILL.md

- **[行 18-24] 新增触发条件**：明确列出 solution figure 的触发关键词（旋转/动点/参数变化/"当...时求"/"使得...成为"），并明确静态题不生成 solution figure。防止对非动画题错误生成 animation 导致 schema 校验失败。（D1, D6）

- **[行 38-43] 新增顺时针旋转公式**：补充"顺时针旋转 = sin 项符号翻转"的等价公式，直接覆盖 geo-010（绕A顺时针旋转β角）的需求。（D3, D6）

- **[行 148-151] 新增"第零步：确保输出完整 JSON"**：强调动画题必须同时输出 geometryFigure 和 solutionGeometryFigure，缺少任一扣分。（D1, D6）
