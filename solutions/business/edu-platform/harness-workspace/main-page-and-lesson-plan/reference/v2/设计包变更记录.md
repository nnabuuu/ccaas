## 一、设计规范.md — 重写

### 色彩系统

| 项目 | v1 | v2 | 原因 |
|------|-----|-----|------|
| 变量命名 | `--bg1/bg2/bg3, --t1/t2/t3, --b1` | `--bg/surface/surface2, --t1/t2/t3, --border` | 语义化命名，更易理解 |
| 页面底色 | `#eeeee8` | `#f4f3ef` | tinted neutral，不是冷灰 |
| 卡片背景 | `#fff`（纯白） | `#fbfaf7` | oiloil guide 禁止纯白 |
| 次级背景 | `#f5f5f0` | `#edece7` | 与新底色匹配 |
| 主文字 | `#1a1a1a` | `#1c1c1a` | 微调，带暖色调 |
| 次级文字 | `#6b6b65` | `#5c5b56` | 加深，保证导航可读性 |
| 边框 | `rgba(0,0,0,.12)` | `rgba(28,28,26,.07)` | 更轻更暖 |
| 语义色 | 7 对（info/success/warn/danger/purple/teal/coral） | 7 对（blue/green/amber/red/purple/teal/coral） | 语义命名更直觉 |

### 字体

| v1 | v2 | 原因 |
|-----|-----|------|
| `-apple-system, "SF Pro Text", "PingFang SC"` | `"Plus Jakarta Sans", -apple-system, "PingFang SC"` | oiloil guide 禁止系统默认字体 |
| 正文主力 11-12px | 正文主力 12-13px | 配合 Plus Jakarta Sans 的字形 |
| 字重两档 400/500 | 字重三档 400/500/700 | hero 标题需要更强层次 |

### 导航系统（新增整节）

v1 没有导航系统规范。v2 新增 §5：

- 响应式双模导航：≥1200px sidebar（232px）/ <1200px top nav（48px）
- Sidebar 完整规范：导航项高度、图标、激活态（左竖线+反色图标）、hover、红点、底部用户信息
- Top nav 规范：链接字号 13px、非激活色 `--t2`、激活态 pill 背景
- 选择 sidebar 的四条决策依据

### 页面布局（新增整节）

v1 定义了三栏/单栏/仪表盘三种布局。v2 重写为 §6：

| v1 | v2 |
|-----|-----|
| 三种静态布局模式 | 四种 max-width 策略（800/860/920/640px）按页面类型分配 |
| 内容居中 | 内容左对齐（紧跟 sidebar 右侧） |
| 无响应式策略 | sidebar + top nav 响应切换 |

### 组件规范

| 组件 | v1 → v2 变化 |
|------|-------------|
| 按钮 | 圆角 8px → 6px |
| 卡片 | `.5px` 边框 → `1px`，圆角 12px → 10px |
| 输入框 | 无 focus 规范 → focus 用紫色半透明 border |

| Section label | 未定义 → 10px uppercase letter-spacing .4-.6px |
| 学业要求锚点 | 不存在 → 新增组件（teal/amber 两种状态） |
| AI 区块 | 不存在 → 新增组件（洞察+chips+输入框一体） |

### 删除的内容

- §8 特殊元素中的 MCP badge 规范（开发者信息不应出现在教师 UI）
- §8 特殊元素中的 @ 引用 pill 规范（移到 Chat 融合入口 PRD）

## 二、PRD-04-首页.md — 新建

全新文档，9 章节。首页之前没有独立 PRD。

核心设计决策：
- 首页回答三个问题：落下什么了（待办）→ 没注意到什么（AI 洞察）→ 最近发生了什么（动态）
- 四个区块信息零重叠
- Focus card：系统事实（截止日期、数量），可展开"还有 N 项"
- AI section：Skill 产出的洞察（趋势、异常、建议），不重复待办内容。包含 suggestion chips + 输入框
- 周视图替代月历，作为时间线的日期筛选器
- Stats 融入问候语，不用独立卡片网格
- 新增 3 个聚合 API，不新增数据表
