# 设计规范

> 成都区教育精准教学平台 · UI 风格指南
> v2.0 · 基于 oiloil-ui-ux-guide 审查后重构

---

## 1. 设计基调

暖灰色系 + 低饱和度语义色。排版驱动层次，留白即设计。

关键词：安静、克制、紧凑、暖色调、排版优先。

不使用：渐变、阴影（box-shadow）、圆角过大的卡片、emoji 作为图标、Inter/Roboto/Arial 字体、纯黑 `#000` / 纯白 `#fff` / 纯灰 `#888`。

---

## 2. 色彩体系

### 2.1 基础色（tinted neutrals）

所有灰色带暖色调，不是标准的冷灰。

| 变量 | 色值 | 用途 |
|------|------|------|
| `--bg` | `#f4f3ef` | 页面底色 |
| `--surface` | `#fbfaf7` | 卡片/面板/sidebar 背景 |
| `--surface2` | `#edece7` | hover 背景、分组底色、输入框底 |
| `--t1` | `#1c1c1a` | 主文字、标题、主按钮背景 |
| `--t2` | `#5c5b56` | 次级文字、描述、非激活导航 |
| `--t3` | `#9c9a92` | 辅助文字、时间戳、占位符 |
| `--border` | `rgba(28,28,26,.07)` | 边框、分割线 |

### 2.2 语义色（低饱和度配对）

每种语义色为一对：深色文字 + 浅色底。

| 语义 | 文字色 | 底色 | 用途 |
|------|--------|------|------|
| 蓝/信息 | `#1a5fa0` | `#e4eff8` | 进行中、作业相关 |
| 绿/成功 | `#2d6612` | `#e6f2dc` | 完成、通过、课堂执行 |
| 琥珀/警告 | `#7a4d0e` | `#f6edda` | 注意、待审批、课标更新 |
| 红/错误 | `#942929` | `#f8e6e6` | 待办紧急度、失败 |
| 紫/AI | `#3a3185` | `#eceafe` | AI 相关、Skill、教案来源 |
| 青/课标 | `#0d5245` | `#ddf1eb` | 学业要求、课标关联 |
| 珊瑚/推优 | `#6b2a14` | `#f7ebe5` | 推优、差异化标注 |

## 3. 字体

### 3.1 字体栈

```css
font-family: "Plus Jakarta Sans", -apple-system, "PingFang SC", sans-serif;
```

Plus Jakarta Sans 作为品牌字体（西文 + 数字），苹方作为中文 fallback。不使用 Inter、Roboto、Arial、Open Sans、system-ui。

### 3.2 字号体系

| 场景 | 字号 | 字重 |
|------|------|------|
| 页面大标题（hero） | 24-28px | 700 |
| 编辑器标题 | 20-22px | 700 |
| 区块标题 / 卡片标题 | 14px | 500 |
| 导航链接 | 13px | 500 |
| 正文 / 列表项 | 12-13px | 400 |
| 辅助信息 / meta | 11px | 400 |
| 标签 / badge | 9-10px | 500-600 |
| section label | 9-10px | 600, uppercase, letter-spacing .4-.6px |

### 3.3 字重

三档：400（正文）、500（导航/副标题）、600-700（标题/强调）。不使用 bold 和 black。

## 4. 间距与圆角

### 4.1 间距系统

基础单位：4px。允许的间距值：4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48。

| 场景 | 间距 |
|------|------|
| 卡片/面板内边距 | 16-20px |
| 列表项间距 | 8px |
| 元素间距（同组） | 4-8px |
| 区块间距（不同功能区） | 24-32px |
| 页面边距 | 24px（窄屏） / 48px（宽屏） |

### 4.2 圆角

| 场景 | 圆角 |
|------|------|
| 卡片 / 面板 / 弹窗 | 10-12px |
| 按钮 / 输入框 | 6-8px |
| badge / pill / 标签 | 3-4px |
| sidebar 导航图标 | 5px |
| 头像 | 6px |

### 4.3 边框

统一 `1px solid var(--border)`。不使用 2px 以上边框。选中/激活状态通过颜色或背景变化表达，不通过加粗边框。

## 5. 导航系统（关键设计决策）

### 5.1 响应式双模导航

平台采用**宽屏 sidebar + 窄屏 top nav**的响应式导航策略。

**≥ 1200px（桌面/超宽）** — 左侧固定 sidebar（232px），top nav 隐藏。

**< 1200px（笔记本/平板）** — sidebar 隐藏，切换为水平 top nav。

```css
.sidebar { display: none; }
@media (min-width: 1200px) { .sidebar { display: flex; } }

.topnav { display: flex; }
@media (min-width: 1200px) { .topnav { display: none; } }
```

### 5.2 Sidebar 规范（宽屏）

| 属性 | 值 |
|------|-----|
| 宽度 | 232px |
| 背景 | `--surface` |
| 位置 | `position: fixed; top: 0; left: 0; bottom: 0` |
| 分组 | "导航" + "管理" 两个 section，用 label 分隔 |
| 导航项高度 | 36px（桌面鼠标精度，不需要 44px 触摸目标） |
| 图标 | 20x20 圆角方块内嵌 12x12 SVG（Lucide 风格线条图标） |
| 激活态 | 左侧 3px 黑色竖线 + `--surface2` 背景 + 图标反色（黑底白 icon） |
| Hover 态 | `--surface2` 背景 |
| 红点 | 右侧 badge，使用 `--red-bg` + `--red` |
| 底部 | 用户信息区：头像（首字圆角方块）+ 姓名 + 角色 |

### 5.3 Top nav 规范（窄屏）

| 属性 | 值 |
|------|-----|
| 高度 | 48px |
| 背景 | `--surface` |
| Logo | 14px, weight 700 |
| 链接字号 | 13px, weight 500 |
| 非激活色 | `--t2`（不是 `--t3`，保证可读性） |
| 激活态 | `--t1` 文字 + `--surface2` pill 背景 |
| 红点 | 同 sidebar |

### 5.4 为什么选 sidebar

决策依据：

1. **模块数量**：7 个导航项 + 管理 + 未来扩展。Top nav 水平空间有限，sidebar 纵向空间无限
2. **宽屏利用率**：640px 内容居中在 2560px 屏上浪费 75% 空间。Sidebar 让导航占据固定空间，内容区充分利用剩余宽度
3. **业界标准**：Linear、Notion、Asana、Vercel 等 SaaS 产品均采用 sidebar 模式
4. **用户角色可见性**：sidebar 底部常驻用户信息（姓名 + 学科 + 年级），top nav 做不到

## 6. 页面布局模式

### 6.1 内容宽度策略

不同页面类型使用不同的 max-width：

| 页面类型 | max-width | 示例 |
|---------|-----------|------|
| 信息流页面（阅读为主） | 800px | 首页 |
| 列表管理页面 | 860px | 教案列表、模板列表 |
| 编辑器页面 | 920px | 教案编辑器（含 200px 侧边栏） |
| 纯编辑页面 | 640px | 模板编辑器 |

内容区在宽屏上**左对齐**（紧跟 sidebar 右侧），不居中。这是 Linear/Notion 的做法——导航在左，工作区紧邻其右，超宽屏右侧留白。

```css
.main {
  margin-left: var(--sidebar-w);  /* 宽屏 */
  padding: 32px 48px 80px;
}
.content { max-width: 800px; }    /* 不加 margin: 0 auto */
```

### 6.2 页面结构模板

```
┌────────────────────────────────────────────────┐
│ [sidebar 232px]  [main content area]           │
│                                                │
│  Logo            内容（max-width 视页面类型）    │
│  ─────           └ 左对齐，右侧自然留白         │
│  导航                                          │
│  · 首页                                        │
│  · 教案 ←act                                   │
│  · 课堂                                        │
│  · 作业 [3]                                    │
│  · 学情                                        │
│  · 资源                                        │
│  ─────                                         │
│  管理                                          │
│  · 管理                                        │
│                                                │
│  ─────                                         │
│  [陈] 陈老师                                    │
│      数学·八年级                                │
└────────────────────────────────────────────────┘
```

### 6.3 编辑器内部布局

教案编辑器内部使用两栏网格：

```css
.ed-layout {
  display: grid;
  grid-template-columns: 1fr 200px;
  gap: 24px;
}
@media (max-width: 800px) {
  .ed-layout { grid-template-columns: 1fr; }
}
```

左栏：标题 → meta → 学业要求锚点 → block 编辑器 → 关联练习 → 操作按钮。
右栏：关联资源 + 模板 + 文件管理。窄屏时右栏收到下方。

## 7. 组件规范

### 7.1 按钮

| 变体 | 样式 |
|------|------|
| 默认 | `border: 1px solid var(--border); background: var(--surface); color: var(--t2)` |
| 主要 | `background: var(--t1); color: var(--surface)` |
| AI | `background: var(--purple-bg); color: var(--purple)` |
| 小尺寸 | `font-size: 11px; padding: 4px 10px` |

圆角 6px。hover 用 `border-color` 变深或 `opacity: .85`。无 box-shadow。

### 7.2 卡片（列表项）

```css
background: var(--surface);
border: 1px solid var(--border);
border-radius: 10px;
padding: 16px 20px;
/* hover: border-color 加深，无 shadow */
```

### 7.3 Badge / 标签

```css
font-size: 10px;
padding: 2px 8px;
border-radius: 4px;
font-weight: 500;
/* 使用语义色配对：background + color */
```

### 7.4 Section label

```css
font-size: 10px;
font-weight: 600;
color: var(--t3);
text-transform: uppercase;
letter-spacing: .4-.6px;
margin-bottom: 12px;
```

### 7.5 输入框

```css
padding: 8px 12px;
border: 1px solid var(--border);
border-radius: 8px;
font-size: 12px;
background: var(--surface);
/* focus: border-color 变为 rgba(58,49,133,.3)（紫色半透明） */
```

### 7.6 Tab 切换

```css
.tab {
  padding: 6px 12px;
  font-size: 11-13px;
  font-weight: 500;
  color: var(--t3);
  border-bottom: 2px solid transparent;
}
.tab.act {
  color: var(--t1);
  border-bottom-color: var(--t1);
}
```

### 7.7 学业要求锚点

教案编辑器的核心组件。两个状态：

**已关联**：teal 底色区块，显示课标编号 + 条目文本 + 区级解读。右侧"更换"链接。

```css
background: var(--teal-bg);
border-radius: 8px;
padding: 14px 16px;
```

**未关联**：amber 虚线框，引导点击关联。

```css
border: 1px dashed var(--amber);
border-radius: 8px;
```

### 7.8 AI 区块

首页的 AI 洞察 + suggestion chips + 输入框，作为一个整体卡片。

- 洞察条目用紫色小圆点（4px，opacity .5）
- Suggestion chips 用 `--purple-bg` + `--purple` 边框
- 输入框在卡片底部，placeholder："或者直接问..."
- "完整对话 ↗" 链接跳转到 Chat 融合入口

## 8. 图标规范

不使用 emoji 作为图标（non-negotiable）。

使用 Lucide 风格的 SVG 线条图标：stroke-width 2, stroke-linecap round。颜色继承 `currentColor`。

sidebar 导航图标：12x12 SVG 嵌在 20x20 圆角方块内。
内联功能图标：14x14。
小装饰图标（如发送按钮内）：14x14。

---

## 9. 交互模式

### 9.1 Hover 反馈

统一使用 `background: var(--surface2)` 或 `border-color` 加深。不使用 opacity 变化（除主按钮外）、不使用 shadow、不使用 scale。

### 9.2 Focus 样式

输入框 focus：`border-color: rgba(58,49,133,.3)` （紫色半透明）。
不使用默认浏览器 outline，用 border-color 替代。

### 9.3 展开/折叠

直接 display 切换（`display: none` → `display: block`）。不使用动画（Phase 1）。

### 9.4 选中状态

- sidebar 导航：左侧竖线 + 背景 + 图标反色
- tab：下划线变黑
- 列表项：border-color 变深
- 周视图日期：黑底白字圆角方块

## 10. 首页特有设计决策

| 决策 | 描述 |
|------|------|
| 信息零重叠 | 四个区块各有唯一职责：focus card = 系统待办，AI section = Skill 洞察，周视图 = 日期导航，timeline = 活动记录 |
| Focus card vs AI section | focus card 是数据库里的客观事实（截止日期、数量），AI section 是 Skill 产出的主观洞察（趋势、异常、建议）。两者不重复同一信息 |
| Stats 融入问候 | 不使用独立的 stats 卡片网格，本周概要融入问候语（"本周编辑了 4 份教案，批改了 67 份答卷"） |
| 日历形态 | 周视图（7 天一行），不是月历网格。点击某天过滤时间线 |
| 快捷操作 | 作为 AI section 的 suggestion chips，不是独立的按钮区块 |
| AI 输入框 | 嵌在 AI 洞察区块内部（洞察 → chips → 输入框），Phase 1 发送后跳转 Chat 融合入口 |

---

## 11. 教案模块特有设计决策

| 决策 | 描述 |
|------|------|
| 学业要求锚点 | 每份教案的第一个可见内容是学业要求（在标题/meta 之后、所有 block 之前）。设计理念：每份教案都是在回应一个具体的学业要求 |
| 列表项信息层次 | 标题 → 学业要求（teal，最显眼）→ meta。未关联的显示 amber 警告 |
| 模板与教案的关系 | Fork 模型：复制 blocks，独立编辑，版本更新提示 |
| 模板编辑器 | 共用 block 编辑器组件，但 block content 为 italic 灰色 placeholder，可标记"建议保留" |
| 教案/模板切换 | 同一模块内用 page-level tab 切换（教案 / 模板），不是独立的导航入口 |
