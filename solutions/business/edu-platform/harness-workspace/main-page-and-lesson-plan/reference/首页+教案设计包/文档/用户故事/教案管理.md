# 教案管理 — 用户故事

> 核心设计理念：每份教案都是在回应一个具体的学业要求。学业要求是锚点，不是附属信息。

### 已确认决策

| 问题 | 决策 |
|------|------|
| 学业要求是否必填 | Phase 1 不强制，但未关联的教案列表中显示 amber 警告，发布时弹确认 |
| 学业要求快照 | 关联时创建快照（code + text + version），课标更新不自动改变教案 |
| 教案创建方式 | 先选学业要求 → 再选模板 → 进入编辑器（学业要求可跳过） |
| 模板和教案的关系 | Fork 模型：复制结构，独立编辑，版本更新提示 |
| 模板推优 | 个人 → 校本 → 区级，逐级审核 |
| 教案和课堂执行 | 教案是课堂执行的可选起点，教师可跳过教案直接设计课堂流 |
| 练习关联 | 教案只关联练习实体，不直接管理题目 |

## Story 1: 关联学业要求

### User Story: 让每份教案有一个明确的教学目标锚点

**Use Case:**
- As a 准备明天 SSS/SAS 判定课的教师
- I want to 先明确这节课要达成的学业要求，再围绕这个目标写教案
- so that 我的教案有据可依，而不是凭感觉设计

**Acceptance Criteria:**

Scenario 1: 新建教案时选择学业要求
- Given: 教师点击"新建教案"
- When: 系统显示学业要求选择面板
- Then: 按学科 → 章节 → 条目三级导航展示课标树，也可搜索关键词。每个条目显示编号、文本、区级解读（如有）。教师选中一条后点"下一步"

Scenario 2: 跳过学业要求
- Given: 教师在学业要求选择面板
- When: 点击"跳过，稍后关联"
- Then: 进入模板选择步骤，教案创建后 requirement_id 为 null

Scenario 3: 编辑器中查看学业要求
- Given: 教案已关联学业要求 "7.3.2 能运用 SSS、SAS 判定三角形全等"
- When: 打开编辑器
- Then: 标题和 meta 下方显示 teal 底的学业要求锚点区块，包含：课标编号、条目文本、区级解读摘要。右侧有"更换"链接

Scenario 4: 编辑器中关联学业要求（补充关联）
- Given: 教案未关联学业要求，编辑器顶部显示 amber 虚线框
- When: 点击虚线框
- Then: 弹出学业要求选择面板，选中后虚线框变为 teal 底的锚点区块

Scenario 5: 课标版本更新
- Given: 教案关联的学业要求区级解读从 v2.0 更新到 v2.1
- When: 教师打开教案编辑器
- Then: 学业要求锚点区块顶部显示一次性横幅"区级解读已更新（v2.0 → v2.1），点击查看变更"。教师可查看差异并确认更新

Scenario 6: 列表中的学业要求展示
- Given: 教师查看教案列表
- When: 浏览各教案
- Then: 已关联学业要求的教案在标题下方显示 teal 色的课标条目文本（带 checkmark 图标）。未关联的显示 amber 色"未关联学业要求"

## Story 2: 从模板创建教案

### User Story: 选好目标，选好结构，只管填内容

**Use Case:**
- As a 已选定学业要求的教师
- I want to 选择一个结构模板快速创建教案
- so that 我把时间花在内容而不是结构设计上

**Acceptance Criteria:**

Scenario 1: 模板选择
- Given: 教师在新建教案流程中已选定学业要求
- When: 进入模板选择步骤
- Then: 显示可用模板列表，按来源分组（区级/校本/我的）。每个模板显示：名称、结构摘要（如"教学目标 → 重难点 → 教学过程 → ..."）、使用次数、适用课型。模板按所选学业要求的学科和课型过滤

Scenario 2: 使用模板
- Given: 教师选择了"新授课标准模板"
- When: 点击"使用此模板"
- Then: 系统创建新教案，模板的 blocks 完整复制，content 为提示文字。source_template_id 记录来源。教师进入编辑器，学业要求锚点已在顶部，下方是模板结构

Scenario 3: 从空白创建
- Given: 模板选择步骤
- When: 点击"从空白创建"
- Then: 创建一个只有学业要求锚点的空白教案，教师从 "+ 添加内容块" 开始

Scenario 4: 模板版本更新
- Given: 教案从"新授课标准模板 v1"创建，模板现已更新到 v2
- When: 教师打开教案编辑器
- Then: 侧边栏"模板"区域显示"来源模板有新版本"，点击可查看结构差异。教师可选择应用更新或忽略

## Story 3: 内容块编辑器

### User Story: 像写笔记一样编辑教案

**Use Case:**
- As a 正在编辑教案的教师
- I want to 用不同类型的内容块自由组装教案内容
- so that 教案结构灵活但有序

**Acceptance Criteria:**

Scenario 1: 插入内容块
- Given: 编辑器中已有若干内容块
- When: hover 到两个块之间，点击出现的"+"
- Then: 弹出内容块类型选择器（章节标题/段落/列表/表格/时间线/提示框/图片），选中后在该位置插入新块并自动获取焦点

Scenario 2: 编辑时间线块
- Given: 教师插入了一个"时间线"块
- When: 编辑教学过程
- Then: 每行显示时间段 + 时长 + 内容描述，可增删行，可拖拽排序。时间段自动根据上一行的结束时间接续

Scenario 3: 拖拽排序
- Given: 编辑器中有多个内容块
- When: 抓住块左侧的 handle 拖动
- Then: 块可以在列表中移动位置，释放后保存新顺序

Scenario 4: 学情提示框
- Given: 编辑器中，教师想记录班级学情
- When: 插入"提示框"类型的块
- Then: 显示 amber 底色的提示框，教师填入学情备注（如"八(2)班 SSS 错误率 42%"）。这个内容会作为 AI 上下文字段供下游 Skill 读取

## Story 4: 管理模板

### User Story: 沉淀和分享好的教案结构

**Use Case:**
- As a 经验丰富的教师
- I want to 把自己常用的教案结构保存为模板，并分享给同事
- so that 其他教师也能受益于我的教案设计经验

**Acceptance Criteria:**

Scenario 1: 从教案保存为模板
- Given: 教师编辑了一份结构完善的教案
- When: 点击"保存为模板"
- Then: 系统提取 blocks 结构（保留 content 为参考），教师填写模板名称和说明，保存为个人模板

Scenario 2: 浏览模板
- Given: 教师进入模板管理页
- When: 查看三个 tab（区级/校本/我的）
- Then: 每个模板卡片显示名称、适用课型、结构摘要、使用次数。区级和校本模板不可编辑，个人模板可编辑/删除

Scenario 3: 编辑模板
- Given: 教师点击自己的个人模板
- When: 进入模板编辑器
- Then: 使用同一个 Block 编辑器组件，但 Block 的 content 是 placeholder 提示文字。每个 Block 可标记"建议保留"。不显示学业要求锚点和练习关联（模板只定义结构）

Scenario 4: 推优提交
- Given: 教师有一个个人模板想分享到校级
- When: 点击"提交推优"
- Then: 填写推荐理由，提交后状态变为"审核中"。校管理员收到推优通知

Scenario 5: 审核推优（管理员视角）
- Given: 校管理员收到模板推优申请
- When: 查看推优详情
- Then: 看到模板结构、使用次数、教师推荐理由。可以批准（发布为校本模板）、驳回（附理由）或建议修改

## Story 5: 教案与课堂执行衔接

### User Story: 教案无缝变成可上课的课堂流

**Use Case:**
- As a 教案写好准备上课的教师
- I want to 把教案一键转换为课堂流
- so that 我不需要在课堂流编辑器中重新录入教学过程

**Acceptance Criteria:**

Scenario 1: 生成课堂流
- Given: 教案状态为"已发布"，包含 timeline 类型的内容块
- When: 点击"生成课堂流"
- Then: 跳转到课堂流编辑器，教案中的 timeline 块自动转换为课堂流的 slide 序列。每个 slide 的 content_description 来自 timeline 的内容描述。学业要求关联自动继承

Scenario 2: 教案没有 timeline 块
- Given: 教案内容中没有 timeline 类型的块
- When: 尝试生成课堂流
- Then: 系统创建空白课堂流并关联教案。教师在课堂流编辑器中手动设计

## Story 6: 教案使用后迭代

### User Story: 教案是活的文档

**Use Case:**
- As a 上完课的教师
- I want to 根据课堂实际情况修改教案
- so that 下次教同一内容时可以用改进后的版本

**Acceptance Criteria:**

Scenario 1: 使用中的教案可编辑
- Given: 教案状态为"使用中"（已关联课堂记录）
- When: 教师修改教案内容
- Then: 修改正常保存，Activity Tracker 记录变更摘要（如"更新了内容块 SAS 判定条件"）

Scenario 2: 课堂反馈触发教案修改
- Given: 教师在课堂记录中写了反思"SAS 夹角概念学生理解困难，需要增加反例"
- When: 教师回到教案
- Then: 侧边栏"关联资源"中可以看到课堂记录的链接，点击查看反思内容，据此修改教案

## 数据源汇总

| Story | 核心数据 | API |
|-------|---------|-----|
| Story 1 | requirement, lesson_plan.requirement_id | `POST /lesson-plans/{id}/link-requirement` |

| Story 3 | lesson_plan.blocks | `POST /lesson-plans/{id}/blocks` |
| Story 4 | template, promotion | `POST /lesson-plans/{id}/save-as-template`, `POST /templates/{id}/promote` |
| Story 5 | lesson_plan → classroom_flow | `POST /lesson-plans/{id}/generate-flow` |
| Story 6 | lesson_plan, classroom_record | `POST /lesson-plans/{id}` |
