# 设计规范

> 成都区教育精准教学平台 · UI 风格指南
> 从 17 个原型中提取的统一设计语言。

---

## 1. 设计基调

暖灰色系 + 低饱和度语义色。整体感受是"安静的工具"——不抢注意力，信息密度高但不拥挤。

关键词：扁平、克制、紧凑、暖色调、无装饰。

不使用：渐变、阴影（box-shadow）、圆角过大的卡片、鲜艳的品牌色、图标字体库。

---

## 2. 色彩体系

### 2.1 基础色（暖灰）

整个界面由三层暖灰构成背景层次，文字使用三级灰度。

| 变量 | 色值 | 用途 |
|------|------|------|
| `--bg1` | `#ffffff` | 卡片/面板背景 |
| `--bg2` | `#f5f5f0` | 次级背景、输入框底、代码块底 |
| `--bg3` | `#eeeee8` | 页面底色、分割区域 |
| `--t1` | `#1a1a1a` | 主文字、标题 |
| `--t2` | `#6b6b65` | 次级文字、描述、标签 |
| `--t3` | `#9c9a92` | 辅助文字、时间戳、占位符 |
| `--b1` | `rgba(0,0,0,.12)` | 边框、分割线 |

注意：不是标准的冷灰（`#f5f5f5`），而是偏暖的灰（`#f5f5f0`）。这个 0 → f0 的微调让整个界面偏暖而非偏冷。

### 2.2 语义色（低饱和度配对）

每种语义色都是一对：浅底色 + 深文字色。用于状态标签、提示条、数据指标。

| 语义 | 底色 | 文字色 | 用途 |
|------|------|--------|------|
| 信息/蓝 | `#E6F1FB` | `#0C447C` | 进行中、链接、一般信息 |
| 成功/绿 | `#EAF3DE` | `#27500A` | 完成、通过、正确 |
| 警告/琥珀 | `#FAEEDA` | `#854F0B` | 注意、待处理、软冲突 |
| 错误/红 | `#FCEBEB` | `#A32D2D` | 失败、拒绝、错误 |
| 紫色 | `#EEEDFE` | `#3C3489` | AI/Skill 相关、品牌强调 |
| 青色 | `#E1F5EE` | `#085041` | 辅助分类、数据可视化 |
| 珊瑚色 | `#FAECE7` | `#712B13` | 辅助分类、差异化标注 |

使用规则：底色只用于小面积元素（标签、pill、提示条）。大面积背景始终使用 bg1/bg2/bg3。

### 2.3 完整色值索引

```
暖灰系列：#ffffff → #f5f5f0 → #eeeee8 → #c8c7c0 → #9c9a92 → #6b6b65 → #1a1a1a
蓝色配对：#E6F1FB / #0C447C
绿色配对：#EAF3DE / #27500A
琥珀配对：#FAEEDA / #854F0B
红色配对：#FCEBEB / #A32D2D
紫色配对：#EEEDFE / #3C3489
青色配对：#E1F5EE / #085041
珊瑚配对：#FAECE7 / #712B13
```

## 3. 字体

### 3.1 字体栈

```css
font-family: -apple-system, "SF Pro Text", "PingFang SC", sans-serif;
```

不引入任何外部字体。依赖系统字体，中文用苹方/思源，西文用 SF Pro/系统默认。

### 3.2 字号体系

原型中字号使用频率（从高到低）：

| 字号 | 频率 | 典型用途 |
|------|------|---------|
| 11px | 最高 | 正文、列表项、表格内容 |
| 10px | 高 | 辅助信息、标签内文字、时间戳、元数据 |
| 12px | 高 | 段落正文、描述文字、输入框 |
| 13px | 中 | 卡片标题、列表项标题 |
| 14-15px | 低 | 页面区块标题 |
| 9px | 低 | 极小标签（pill 内文字、脚注） |
| 18-20px | 低 | 页面主标题、数据大数字 |
| 28-48px | 极低 | 仪表盘核心指标数字 |

注意：整体字号偏小。11-12px 是正文主力，不是 14-16px。这是信息密度优先的设计选择。

### 3.3 字重

只使用两档：
- `400`（regular）— 正文、描述
- `500-600`（medium/semibold）— 标题、标签、强调

不使用 `700`（bold）。

## 4. 间距与圆角

### 4.1 圆角

| 变量 | 值 | 用途 |
|------|-----|------|
| `--r` | `8px` | 通用圆角（按钮、输入框、小卡片） |
| `--rl` | `12px` | 大卡片、面板、弹窗 |
| `3-4px` | — | 标签 pill、小徽章 |
| `50%` | — | 圆形头像、状态点 |
| `6px` | — | 内嵌区块、代码块 |

### 4.2 间距

紧凑排列。核心间距值：

| 场景 | 间距 |
|------|------|
| 卡片内边距 | `12-16px` |
| 列表项内边距 | `6-10px` |
| 元素间距（同组） | `4-8px` |
| 区块间距 | `10-16px` |
| 导航项高度 | `约 30-36px` |

### 4.3 边框

统一使用 `.5px` 或 `1px` 边框，颜色为 `--b1`（`rgba(0,0,0,.12)`）。

不使用 `2px` 以上的边框。选中/激活状态通过颜色变化表达，不通过加粗边框。

## 5. 组件规范

### 5.1 按钮

```css
/* 默认按钮 */
.btn {
  font-size: 12px;
  padding: 6px 14px;
  border-radius: 8px;
  border: .5px solid rgba(0,0,0,.12);
  background: #fff;
  color: #6b6b65;
}

/* 主要按钮 */
.btn.pri {
  background: #1a1a1a;
  color: #fff;
  border-color: #1a1a1a;
}

/* AI 按钮 */
.btn.ai {
  background: #EEEDFE;
  color: #3C3489;
  border-color: transparent;
}

/* 小尺寸 */
.btn.sm {
  font-size: 11px;
  padding: 4px 10px;
}
```

规则：
- 主要操作用黑底白字（`.pri`），不用品牌色
- AI 相关操作用紫色底（`.ai`）
- 破坏性操作用文字变红，不用红色底
- 按钮无 box-shadow

### 5.2 标签（Badge / Pill）

```css
.badge {
  font-size: 9-10px;
  padding: 2px 6-8px;
  border-radius: 3-4px;
  font-weight: 500;
}
```

语义标签用对应的底色+文字色配对：
- 运行中 → 蓝底蓝字
- 已完成 → 绿底绿字
- 待审批 → 琥珀底琥珀字
- 已驳回 → 红底红字
- Skill 标注 → 紫底紫字

### 5.3 卡片

```css
.card {
  background: #fff;
  border: .5px solid rgba(0,0,0,.12);
  border-radius: 12px;
  overflow: hidden;
}
```

卡片内部用 `border-bottom` 分割区块。不使用内部阴影或嵌套卡片阴影。

### 5.4 导航栏（页面级）

```css
.page-nav {
  position: sticky;
  top: 0;
  background: #fff;
  border-bottom: .5px solid rgba(0,0,0,.12);
  padding: 8px 16px;
  display: flex;
}

.page-nav-btn {
  padding: 6px 12px;
  font-size: 11px;
  border: none;
  background: transparent;
  color: #9c9a92;
  border-bottom: 2px solid transparent;
}

.page-nav-btn.act {
  color: #1a1a1a;
  border-bottom-color: #1a1a1a;
}
```

规则：导航项用下划线指示选中，不用背景色。

### 5.5 侧边栏导航

```css
.nav-item {
  padding: 7px 14px;
  font-size: 12px;
  color: #6b6b65;
}

.nav-item.act {
  background: #1a1a1a;
  color: #fff;
  margin: 1px 6px;
  border-radius: 6px;
}
```

规则：选中项用黑底白字胶囊。计数器置于右侧，选中时变半透明白色。

### 5.6 表单元素

```css
input, select, textarea {
  padding: 6px 8px;
  border: .5px solid rgba(0,0,0,.12);
  border-radius: 6px;
  font-size: 11-12px;
  font-family: inherit;
  background: #fff;
}
```

规则：输入框高度紧凑。select 使用自定义下拉箭头（隐藏原生 appearance）。

### 5.7 信息区块

```css
/* 内嵌信息块（用于输出展示、摘要等） */
.info-block {
  padding: 6-8px;
  background: #f5f5f0;
  border-radius: 6px;
  font-size: 10-11px;
  color: #6b6b65;
}
```

### 5.8 Skill 标注

所有 AI 产出内容标注 Skill 来源，使用紫色 pill：

```css
.skill-pill {
  font-size: 9px;
  padding: 1-2px 5-6px;
  border-radius: 6px;
  background: #FAEEDA;  /* 琥珀底 */
  color: #854F0B;
}
```

## 6. 布局模式

### 6.1 三栏布局（管理类页面）

```
┌─────────────────────────────────────────┐
│ 顶部导航（sticky）                        │
├──────┬──────────────────┬───────────────┤
│ 侧边 │ 主内容区          │ 详情面板       │
│ 导航 │ (网格/列表/表格)   │ (选中项详情)   │
│ 200px│ flex:1            │ 300-340px     │
└──────┴──────────────────┴───────────────┘
```

用于：教案管理、互动模板管理、课标管理、题库管理。

### 6.2 单栏布局（操作类页面）

```
┌─────────────────────────────────────────┐
│ 顶部导航（sticky）                        │
├─────────────────────────────────────────┤
│ 内容卡片（max-width: 640-720px, 居中）     │
│                                         │
│ 内容卡片                                 │
│                                         │
│ 内容卡片                                 │
└─────────────────────────────────────────┘
```

用于：Chat 融合入口、Skill 输出详情、审批操作。

### 6.3 仪表盘布局

```
┌─────────────────────────────────────────┐
│ 顶部导航 + 筛选条件栏                      │
├──────────┬──────────┬───────────────────┤
│ 指标卡片  │ 指标卡片  │ 指标卡片           │
├──────────┴──────────┴───────────────────┤
│ 大图表/表格区域                            │
├──────────────────┬──────────────────────┤
│ 列表/数据块       │ 列表/数据块            │
└──────────────────┴──────────────────────┘
```

用于：管理员概览与分析、学情分析。

### 6.4 原型导航（多视图切换）

原型文件内通过顶部 tab 栏切换多个视图。这是原型展示用的模式，不是产品 UI：

```css
/* 原型内的视图切换 */
nav {
  position: sticky;
  top: 0;
  display: flex;
  gap: 2px;
  overflow-x: auto;
}
```

## 7. 交互模式

### 7.1 展开/折叠

列表项通过点击 header 展开详情。不使用动画，直接 display 切换。

```javascript
function toggle(el) {
  el.nextElementSibling.classList.toggle('open');
}
```

### 7.2 选中状态

- 列表项选中：黑底白字胶囊（侧边栏）或蓝底+左侧竖线（表格行）
- 卡片选中：边框变紫色 + 1px 外描边
- 标签页选中：下划线变黑

### 7.3 悬停状态

统一使用 `background: var(--bg2)` 作为 hover 反馈，不使用 opacity 变化或阴影。

## 8. 特殊元素

### 8.1 AI 产出标注

所有 AI 产出内容带来源标注：

```
┌────────────────────────────────────┐
│ [Skill: 作业综合分析 v1.0]          │  ← 紫色 pill
│                                    │
│ 分析结果内容...                     │
│                                    │
│ [追问 AI ->]   [反馈/优化 Skill]     │  ← 操作按钮
└────────────────────────────────────┘
```

### 8.2 @ 引用 Pill

Chat 输入框中的实体引用显示为带图标的 pill：

```
文字样式：10px, 紫色底 (#EEEDFE), 紫色字 (#3C3489)
结构：[图标] [实体名] [x关闭]
圆角：4px
```

### 8.3 数据指标卡

```
┌──────────────┐
│ 标签 (9px t3) │
│ 数值 (18-28px t1, font-weight:600-700) │
│ 变化 (9px, 绿色涨/红色跌) │
└──────────────┘
背景：var(--bg2)
圆角：6px
内边距：8px
```

### 8.4 状态流转

使用语义色标签表达实体状态：

```
草稿 → 进行中 → 已完成
(灰底)  (蓝底)   (绿底)

待审批 → 已通过 / 已驳回
(琥珀底)  (绿底)   (红底)
```

---

## 9. 不做的事

- 不使用 box-shadow（全部用边框替代）
- 不使用渐变背景
- 不使用图标字体库（用文字或简单字符替代）
- 不使用 emoji 作为功能图标（原型中偶有使用，产品中应替换为 SVG 图标）
- 不使用 hover 时的 scale 变换
- 不使用全屏模态弹窗（用侧边面板或内嵌展开替代）
- 不使用颜色作为唯一的信息区分方式（始终辅以文字标签）
- 不使用大于 12px 的圆角（保持克制）

## 10. CSS 变量速查

```css
:root {
  /* 背景 */
  --bg1: #fff;
  --bg2: #f5f5f0;
  --bg3: #eeeee8;

  /* 文字 */
  --t1: #1a1a1a;
  --t2: #6b6b65;
  --t3: #9c9a92;

  /* 边框 */
  --b1: rgba(0,0,0,.12);

  /* 语义色 */
  --info-bg: #E6F1FB;     --info-t: #0C447C;
  --success-bg: #EAF3DE;  --success-t: #27500A;
  --warn-bg: #FAEEDA;     --warn-t: #854F0B;
  --danger-bg: #FCEBEB;   --danger-t: #A32D2D;

  /* 品牌/分类色 */
  --purple-bg: #EEEDFE;   --purple-t: #3C3489;
  --teal-bg: #E1F5EE;     --teal-t: #085041;
  --coral-bg: #FAECE7;    --coral-t: #712B13;

  /* 圆角 */
  --r: 8px;
  --rl: 12px;
}
```
