# PRD-04 首页

> 版本: 1.0
> 更新日期: 2025-03-23
> 前置文档: `总体架构.md`（产品哲学）、`内部技术架构.md`（Activity Tracker）
> 参考原型: `原型/首页/首页.html`

---

## 1. 概述

### 1.1 系统定位

首页是教师每天打开平台后的第一个页面。它回答三个问题：

1. 我落下什么了？（待办事项）
2. 有什么我没注意到的？（AI 洞察）
3. 最近发生了什么？（活动动态）

首页不是"所有模块的入口大厅"——模块导航由顶栏承担。首页的职责是帮助教师在 10 秒内决定"接下来做什么"。

### 1.2 设计原则

- **信息零重叠**：每个区块有且只有一个职责，不重复展示同一类信息
- **系统事实 vs AI 洞察 分离**：待办事项来自数据库的客观状态（截止日期、提交数量），AI 洞察来自 Skill 的分析产出（趋势、异常、关联建议）
- **单栏流式布局**：不分栏，从上到下一列流下来，max-width 640px，信息密度适中
- **AI 是页面内容的一部分**：洞察和输入框作为一个整体区块嵌在页面正文中，不是浮窗或弹出组件

### 1.3 用户角色

| 角色 | 首页看到的内容 |
|------|-------------|
| 教师 | 个人待办 + AI 洞察（基于个人教学数据）+ 个人活动动态 |
| 校管理员 | 校级待办（审批）+ AI 洞察（校级学情异常）+ 校级活动动态 |
| 区管理员 | 区级待办（审批、推优）+ AI 洞察（区级数据分析）+ 区级活动动态 |

本 PRD 以教师角色为主线描述，管理员角色的差异在各区块中标注。

### 1.4 Phase 范围

| 功能 | Phase 1 | Phase 2 |
|------|---------|---------|
| 问候 + 周概要 | 静态问候 + 本周统计 | 个性化问候（基于教学节奏） |
| Focus card | 按截止日期排序的待办 | 按紧急度 + AI 预测优先级排序 |
| AI 洞察 | 展示最近的 Skill 输出摘要 | AI 主动生成每日 briefing |
| Suggestion chips | 固定常用操作 + 洞察关联操作 | 基于上下文动态生成 |
| 输入框 | 跳转到 Chat 融合入口 | 首页内完成简单对话 |
| 周视图 | 展示 Activity Tracker 数据 | 展示课程安排（对接教务系统） |
| 动态时间线 | 按日期过滤的 Activity 列表 | 支持按实体类型过滤 |

## 3. 区块 A：问候 + 周概要

### 3.1 问候语

根据当前时间显示"早上好 / 下午好 / 晚上好" + 教师姓名。

### 3.2 周概要

一行文字融入问候下方，展示本周的关键统计。数据来源为 Activity Tracker 按周聚合。

格式：`{日期} {星期} · 本周编辑了 {N} 份教案，批改了 {M} 份答卷`

统计维度：
- 教案编辑次数（action = created | updated，entityType = lesson_plan）
- 答卷批改数量（action = updated，entityType = submission）

如果本周无活动，显示 `{日期} {星期}` 不显示统计。

### 3.3 API

```
GET /context/activity/weekly-summary?user_id={uid}
→ { lesson_plan_edits: number, submissions_graded: number, ... }
```

复用 Activity Tracker 的聚合查询能力，不需要新建数据表。

## 4. 区块 B：Focus Card（待办事项）

### 4.1 主卡片

展示当前最紧急的一条待办事项。"最紧急"的排序规则：

1. 截止日期最近的优先
2. 同一天截止的，已提交数/总数比例最高的优先（说明快可以开始处理了）
3. 审批类待办优先于批改类（审批阻塞他人工作）

| 字段 | 说明 |
|------|------|
| 标签 | "需要处理"（固定文案） |
| 标题 | `{班级} {作业名}` — `{N}` 份待批改 / `{申请人}推优申请` — `{N}` 项待审核 |
| 元信息 | 截止时间 · 进度（如 "已提交 32/38"）· Skill 状态（如 "Skill 已预分析完成"） |
| 主按钮 | "去批改" / "去审核"，点击跳转到对应模块页面 |

左侧红色竖线标识紧急性。

### 4.2 折叠的其余待办

主卡片下方显示 "还有 N 项待处理"，点击展开剩余待办列表。每项显示标题 + 截止时间，不显示主按钮（点击整行跳转）。

### 4.3 空状态

无待办时不显示 Focus card 区块。页面从问候直接进入 AI 区块。

### 4.4 数据来源

待办不是一个独立的"待办表"——它从各业务模块实时查询汇总：

| 待办类型 | 数据源 | 条件 |
|---------|--------|------|
| 作业待批改 | `homework` + `submission` | homework.status = 'submitted'，统计 submission 中 status = 'pending' 的数量 |
| 推优待审核 | `proposal` | proposal.status = 'pending'，当前用户是审批人 |
| 课标更新待确认 | `requirement_update` | update.acknowledged = false，当前用户在影响范围内 |

### 4.5 API

```
GET /dashboard/pending?user_id={uid}&limit=5&sort=urgency
→ {
    items: [
      { type: 'grading', title: '八(2)班 SAS 专项练习', count: 32,
        deadline: '2025-03-18', progress: '32/38',
        skill_status: 'analyzed', link: '/homework/hw_1/grade' },
      ...
    ],
    total: 3
  }
```

## 5. 区块 C：AI 洞察 + 输入

### 5.1 定位

展示 AI（Skill）产出的洞察——教师自己看数据不容易发现的趋势、异常和关联。

与 Focus card 的区别：Focus card 是"系统告诉你必须做的事"，AI 洞察是"AI 告诉你可能没注意到的事"。两者零重叠。

### 5.2 洞察内容

来源：最近 24 小时内有新产出的 Skill 输出，按相关性排序取 top 3。

每条洞察包含：
- 一句话摘要（如 "八(2)班 SAS 正确率连续 3 周下降"）
- 关键数据（如 "72% → 64% → 56%"）
- 可操作的建议（如 "建议检查前置知识掌握情况"）

不显示 Skill 名称和运行时间——这些信息属于"AI 产出"详情页，不是首页需要的。

### 5.3 Suggestion Chips

洞察下方显示一行可点击的 chip 按钮，分两类：

| 类型 | 来源 | 示例 |
|------|------|------|
| 上下文关联操作 | 基于上方洞察内容生成 | "分析夹角错因"、"对齐课标 v2.1"、"查看王老师课堂流" |
| 通用快捷操作 | 固定列表，按使用频率排序 | "新建教案"、"发布作业" |

Phase 1：上下文关联操作由后端从 Skill 输出的 `suggested_actions` 字段读取；通用操作为固定列表。

Phase 2：全部由 AI 根据当前上下文动态生成。

点击 chip 的行为：跳转到 Chat 融合入口，chip 文本作为首条消息自动发送。

### 5.4 输入框

洞察和 chips 下方紧跟一个输入框。placeholder："或者直接问..."

Phase 1 行为：用户输入后跳转到 Chat 融合入口，输入内容作为首条消息带过去。首页本身不处理对话。

Phase 2 行为：首页内完成简单的单轮对话（如查询类问题），复杂对话仍跳转。

输入框右侧有"完整对话"链接，点击直接跳转到 Chat 融合入口（无需先输入）。

### 5.5 空状态

如果最近 24 小时没有新的 Skill 输出：

- 标题改为 "AI 助手"（不显示"发现了几件事"）
- 洞察区域显示 "暂无新发现。你可以问我任何问题。"
- Chips 只显示通用快捷操作
- 输入框正常可用

### 5.6 API

```
GET /dashboard/ai-briefing?user_id={uid}&limit=3
→ {
    insights: [
      { summary: '八(2)班 SAS 正确率连续 3 周下降（72%→64%→56%）...',
        source_skill_run_id: 'sr_1',
        suggested_actions: [
          { label: '分析夹角错因', prompt: '详细分析八(2)班 SAS 专项练习中夹角概念混淆的错因' },
          { label: '对齐课标 v2.1', prompt: '帮我检查 SSS/SAS 教案是否符合课标 v2.1 的新要求' },
        ]
      },
      ...
    ],
    common_actions: [
      { label: '新建教案', prompt: '帮我新建一份教案' },
      { label: '发布作业', prompt: '帮我发布一份作业' },
    ]
  }
```

## 6. 区块 D：周视图 + 动态时间线

### 6.1 周视图

一行 7 天，显示本周每天的活动情况。

每天显示：星期 + 日期 + 活动色点（最多 4 个，颜色对应实体类型）。

颜色映射：

| 实体类型 | 色点颜色 |
|---------|---------|
| lesson_plan | purple |
| homework / submission | blue |
| session（课堂执行） | green |
| requirement（课标） | amber |
| classroom_record | teal |
| proposal（推优） | coral |

选中某天：该天日期高亮（黑底白字），下方时间线过滤为该天的活动。默认选中今天。

### 6.2 动态时间线

展示选中日期的所有 Activity 记录，按时间倒序。

每条显示：
- 色点（对应实体类型）
- 一行文字：`{实体名} {动作描述}`
- 时间（相对时间或绝对时间）

动作描述映射：

| action | 描述模板 |
|--------|---------|
| created | 创建了 {entity_display_name} |
| updated | 更新了 {entity_display_name}（如有变更摘要则附加） |
| viewed | 查看了 {entity_display_name} |
| referenced | 在对话中引用了 {entity_display_name} |
| submitted | 收到 {count} 份提交 |

### 6.3 空状态

选中某天无活动时：时间线区域显示 "这一天没有活动记录"。

### 6.4 API

复用 Activity Tracker 的已有 API：

```
GET /context/activity?user_id={uid}&date=2025-03-17&limit=20
→ {
    items: [
      { entity_type: 'lesson_plan', entity_id: 'lp_1',
        entity_display_name: 'SSS/SAS 新授课教案',
        action: 'updated', detail: '更新了内容块 "SAS 判定条件"',
        timestamp: '2025-03-17T14:45:00Z' },
      ...
    ]
  }
```

周视图的色点数据：

```
GET /context/activity/week-dots?user_id={uid}&week_start=2025-03-10
→ {
    days: {
      '2025-03-10': ['lesson_plan'],
      '2025-03-12': ['lesson_plan', 'homework', 'proposal'],
      '2025-03-17': ['lesson_plan', 'homework', 'session'],
      ...
    }
  }
```

## 7. 顶栏导航

顶栏不属于首页特有，但首页原型中定义了其结构：

| 元素 | 说明 |
|------|------|
| Logo | "精准教学"，点击返回首页 |
| 模块链接 | 首页（当前高亮）、教案、课堂、作业、学情、资源、管理 |
| 红点 | 有待办的模块显示数字红点（如 "作业 3"） |

导航链接按使用频率排列。管理员角色可能看到更多链接（如"审批"）。

## 8. 数据依赖

首页不拥有自己的数据表，完全依赖现有系统的数据：

| 区块 | 数据源 | 是否需要新 API |
|------|--------|--------------|
| 问候 | 用户信息 + 当前时间 | 否 |
| 周概要 | Activity Tracker 聚合 | 是：weekly-summary 聚合接口 |
| Focus card | homework + submission + proposal | 是：dashboard/pending 汇总接口 |
| AI 洞察 | SkillRun 输出 | 是：dashboard/ai-briefing 接口 |
| 周视图 | Activity Tracker 按日分组 | 是：week-dots 聚合接口 |
| 动态 | Activity Tracker | 否：复用 /context/activity |

新增 3 个聚合 API，不新增数据表。

## 9. 性能要求

| 指标 | 目标 |
|------|------|
| 首屏加载（问候 + focus card + AI 洞察可见） | < 1s |
| 周视图 + 动态加载 | < 500ms |
| 切换周视图日期 → 时间线刷新 | < 300ms |

Focus card 和 AI 洞察可以并行请求。周视图的 week-dots 可以在首屏渲染后异步加载。
