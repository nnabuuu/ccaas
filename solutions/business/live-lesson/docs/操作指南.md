# Live Lesson 操作指南

## 启动服务

```bash
# 1. 启动后端 (端口 3007)
cd solutions/business/live-lesson/backend
npx nest build && PORT=3007 node dist/main.js

# 2. 启动前端 (端口 5283)
cd solutions/business/live-lesson/frontend
npx vite --port 5283
```

服务就绪后访问 http://localhost:5283

## 教师操作流程

### 第一步：创建课堂

打开 http://localhost:5283，看到课程卡片（如 "Ideal Beauty — 阅读策略训练"），点击 **「创建课堂」**。

![课程选择页](../../../course-selection-page.png)

### 第二步：等待学生加入

系统自动生成一个 **6 位课堂码**（如 `MPD6SU`），投屏或口头告知学生。页面实时显示已加入的学生列表。

![教师等待页](../../../teacher-waiting-page.png)

把课堂码告诉学生，让他们访问 `/join` 页面输入。学生加入后，名字会实时出现在列表中。

![学生已加入](../../../teacher-student-joined.png)

### 第三步：开始上课

确认学生都到齐后，点击 **「开始上课」**（也可以先开始，学生随时可以加入）。

### 第四步：课堂观察台

进入教师仪表盘后，可以看到：

![教师仪表盘](../../../teacher-dashboard-active.png)

**顶部 Health Cards（4 张卡片）**：
- **最快进度** — 跑得最快的学生到了第几步
- **中位进度** — 全班的中位水平
- **卡点学生** — 有多少人卡住了、卡在哪一步
- **AI 对话** — 全班累计 AI 对话轮数和使用人数

**中部 Step 卡片（5 个步骤）**：
1. 图式激活 (Predicting) — 5 min
2. 结构解码 (Skimming) — 8 min
3. 矩阵构建 (Scanning) — 15 min
4. 批判质疑 (Evaluating) — 12 min
5. 复盘升华 (Synthesizing) — 5 min

每个 Step 卡片显示：
- 当前在做这一步的人数 + AI 使用轮数
- 正确率
- 学生头像点阵（绿=完成, 蓝=进行中, 橙=卡住, 灰=阅读中）
- 红色告警标签（如 "Q1 错误偏高"、"5 人卡住"）

**右侧问题队列**：
- 学生通过 AI 面板提出的问题，按分类（概念理解、阅读策略、课文内容、解题求助）归组
- 教师可以看到哪些问题是高频问题，决定是否全班讲解

**底部观察要点**：
- 系统自动识别教学模式和卡点，给出教学建议

### 教师在课堂中关注什么

| 时机 | 看什么 | 做什么 |
|------|--------|--------|
| 开课 5 min | 最快进度 + 中位进度 | 确认全班都进入了 Task 1 |
| 某步耗时过长 | 卡点学生数 | 如果 > 5 人卡住，考虑全班暂停讲解 |
| 某维度错误偏高 | Step 卡片的告警标签 | 点开详情看常见错误，针对性讲解 |
| AI 对话激增 | AI 对话卡片 | 检查问题队列，看学生在问什么 |
| 全班完成某步 | Step 完成率 100% | 口头推进到下一步（或让学生自主推进） |

## 学生操作流程

### 第一步：输入课堂码

打开 http://localhost:5283/join，输入老师给的 **6 位课堂码**，点击「下一步」。

![输入课堂码](../../../student-join-code.png)

### 第二步：输入姓名

确认课程名称正确后，输入自己的姓名，点击 **「加入课堂」**。

![输入姓名](../../../student-join-name.png)

### 第三步：阅读 + 做题

进入学习界面后，屏幕分为两栏：

![学生学习界面](../../../student-waiting.png)

**左栏 — 任务面板**：
- 顶部进度条显示 5 个任务步骤（Predict → Skim → Scan & Build → Evaluate → Wrap-up）
- 当前任务的介绍和说明
- 点击 **「Start Task 1 →」** 开始做题

**右栏 — 课文原文**：
- 完整的英文课文（带段落编号 ¶1, ¶2...）
- 关键信号词高亮（如 "change over time", "different cultures around the world"）
- 随着任务切换，自动聚焦到当前任务相关的段落

**题型包括**：
- **Quiz** — 选择题（有英文和中文提示可用）
- **Match** — 配对题（拖拽或选择匹配）
- **Matrix** — 矩阵填表（Where/What/Why 等维度）
- **Stance** — 观点+证据（选择立场并找到文本证据）
- **Order** — 排序题

### 第四步：AI 助手

右下角有一个 **「?」** 按钮，点击打开 AI 对话面板：
- 可以选择预设问题快速提问
- 也可以自由输入问题
- AI 会根据当前任务和课文内容回答
- 对话记录老师可以在观察台看到

### 学生操作要点

| 步骤 | 操作 | 注意 |
|------|------|------|
| 加入 | 输码 → 输名 → 加入 | 码区分大小写 |
| 做题 | 读题 → 看右栏课文 → 选答案 → 提交 | 每步完成后自动进入下一步 |
| 求助 | 点右下角 ? → 选择或输入问题 | AI 回答后可以继续追问 |
| 提示 | 做题时点"提示"按钮 | 先看英文提示，再看中文提示 |

## 三端关系

```
教师端 (/teacher/:lessonId)          学生端 (/join → 学习界面)
┌──────────────────────┐          ┌──────────────────────┐
│  课堂观察台           │          │  左: 任务面板         │
│  · Health Cards      │◄── SSE ──│  右: 课文原文         │
│  · Step 卡片 + 学生   │  实时同步 │  AI: 右下角对话       │
│  · 问题队列          │          │                      │
└──────────────────────┘          └──────────────────────┘
         │                                  │
         │         POST /submit             │
         │         POST /ai/ask             │
         └──────── GET /stream ─────────────┘
                   (SSE 实时流)
```

- **教师端**：只读观察，不直接操作学生端内容
- **学生端**：独立做题，提交后数据实时同步到教师端
- **投屏板书** (`/board/:lessonId`)：可选，用于教室投影展示结构化板书
