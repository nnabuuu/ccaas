左列 · 中间下拉 · 右列固定选项
1

读两列内容

左列是"题"（词 / 概念 / 现象），右列是"答"（定义 / 解释 / 结果）。两列数量一一对应。

2

建立映射

点击左侧某项 → 弹出右侧候选下拉框；选中一个 = 建立配对。可以反复改。

3

检查全部连完

提交按钮只在所有配对都连上之后才可点击。漏一个都不行。

4

看每对的对错

提交后，每对单独标 ✓ 或 ✗。错的可以重连后再次提交。

03 —

教师能观察到什么 From the Observation Panel

配对级 · 维度统计
每对

哪对最容易错

每个配对单独的正确率。如果"vanity" 这一对全班错了一大半 —— 说明这个词大家都不熟，下一步重点讲它。

错配模式

常见的"错配组合"

"stunning" 经常被配到 "extreme pride" —— 说明学生把它和 "vanity" 混淆了。错配模式比单纯的对错更有诊断价值。

完成时间

快 vs 慢的学生

10 秒钟连完 = 大概率瞎连；3 分钟才提交 = 在真的思考。两者答案都对，但掌握深度不同，需要不同处理。

尝试次数

谁连了几次才全对

一次过 = 单词都熟。三次才过 = 至少一组词有混淆。系统记录每次提交时的错配组合，能精准定位混淆点。

什么时候用 When to Reach for This

✓ 适合✗ 不适合 / 用别的
词汇练习（词 ↔ 定义、词 ↔ 反义词）词义辨析需要例句上下文 —— 用 quiz
概念配对（事件 ↔ 时间、人物 ↔ 作品、原因 ↔ 结果）概念有多对多关系（一个原因对多个结果）—— 用 matrix
对照学习（中文术语 ↔ 英文术语 / 旧概念 ↔ 新概念）开放回答（"为什么这样配？"）—— 用 discuss
检查一组相近词的辨析能力顺序关系（不是配对而是排序）—— 用 order
warmup / 复习课（快速激活旧知识）需要写出定义 —— 用 fill-blank
05 —

出一道题需要准备什么 Authoring Checklist

出题清单

配对题看起来简单，但左右两列怎么设计决定了它是"识记题"还是"辨析题"。

1
决定要配对的"维度"
词 ↔ 定义？事件 ↔ 时间？文化 ↔ 国家？这是题的灵魂。明确维度后再选 4-6 对内容。
2
控制对数：4-6 对最佳
少于 3 对太简单。多于 7 对学生会"猜中了一半就放弃"。5 对是甜点。
3
右列内容要"听起来都合理"
最容易出错的地方：右列答案太特异，学生一眼能看出"这肯定是配那个的"。好的右列让每一项都可能是任何一个左列的答案，迫使学生真的思考。
4
（可选）每对加 hint
如果某对涉及生僻知识，给一行提示。学生卡住时能用，不用就不影响节奏。
核心教学原则

配对题的难度
在右列。

左列定主题，右列定难度。当右列的每一项对任何左列都"看起来合理"时，学生不能靠"看起来像就连"，必须真的理解每对的精确含义。这就是配对题超越选择题的地方 —— 错一个连锁错好几个，逼着学生认真对待每一对。

06 —

常见疑问 FAQ

问题答案
左右列数量必须一样吗？是。1:1 严格对应。如果想出"一对多" / "多余干扰项"，用 quiz 多选。
顺序会随机化吗？每次加载右列顺序会打乱（防止学生记位置）。左列固定。
可以拖拽吗？当前是点击+下拉选择。拖拽交互在路线图上。
错的可以单独重连吗？可以。重新打开任意一行 → 选新答案 → 再次提交。
每对能加图吗？当前左右两列是纯文本。需要图配文，等扩展版本。
能做"多对多"配对吗？不支持。多对多关系本质是 matrix（行列交叉表）的场景。
已落地课程 · In Use
英语词汇课、文言文实词配对、生物学名 ↔ 中文名等场景常用。在 warmup 步骤效果最好。
同类题型 · Related
需要多维交叉关系？用 matrix。需要顺序关系？用 order。需要写答案？用 fill-blank。
出题陷阱 · Pitfalls
最常见错误：(1) 右列答案太特异（一眼看出对应）；(2) 对数太多导致疲劳；(3) 左右列不平衡（左列短词、右列长定义混搭让认知负担集中在右列）。
