Match
左右配对题

左列固定项 + 右列选项池，学生为每个左项选择匹配的右项。case-insensitive 字符串比较，O(pairs) 同步评分。
典型用法：单词配释义、人物配事件、原因配结果。

查看源码 → type: "match"
01 —

Answer Key Schema Zod

服务端结构（含答案） 
const MatchAnswerItemSchema = z.object({
  pairIdx:        z.number(),
  left:           z.string().min(1),
  correct:        z.string().min(1),
  options:        z.array(z.string()).min(2).optional(),
  hint:           z.string().optional(),
  walkthrough:    z.string().optional(),
  paraRef:        z.array(z.number().int().positive()).optional(),
});

const MatchAnswerKeySchema = z.object({
  type:    z.literal('match'),
  answers: z.array(MatchAnswerItemSchema).nonempty(),
  options: z.array(z.string()).min(2).optional(),  // 共享选项池
}).refine(
  (ak) => ak.answers.every((a) => a.options || ak.options),
  { message: 'match: each answer must have options at answer-level or top-level' },
);

字段说明 Fields

字段类型用途
type"match"固定字面量
answers[].pairIdxnumber配对的顺序索引（0-based）
answers[].leftstring左列固定项（学生不操作，只看）
answers[].correctstring正确的右列匹配项（大小写不敏感）
answers[].optionsstring[]?该题独立的选项池（覆盖顶层 options）
options（顶层）string[]?所有题共享的选项池
answers[].hintstring?答错提示
answers[].paraRefnumber[]?关联段落号

04 —

Sanitize / Grade 行为 Plugin Methods

sanitize

剥离哪些字段

从 answerKey 生成 ExerciseSpec，剥离 correct。保留 left / options / paraRef。每个 pair 自动 fallback 到顶层 options。

// sanitize 后学生看到的
{
  type: "match",
  pairs: [
    { idx: 0, left: "...", options: [...] }
  ]
}
grade

评分规则

大小写不敏感字符串比较：student.toLowerCase() === correct.toLowerCase()。支持 data.pairs 或 data.answers（兼容），元素可以是字符串或 { value } 对象。

// 输出
{
  total: 75,
  byDimension: { p0: true, p1: true, p2: false, p3: true }
}

什么时候用 Match When to Use

适合场景不适合场景
单词配释义 / 概念配例子 需要表达推理过程（用 matrix / stance）
历史事件配时间 / 地点 左项的"正确匹配"有多个（用 select-evidence）
因果关系配对 排序型题目（用 order）
选项数量小（2-6 个） 开放式回答（用 image-upload / stance）
迁移状态 · Status
已迁移至 plugin 模式（Stage 1）。Parity tests 验证与 legacy MatchGrader 一致。
特殊点 · Quirks
case-insensitive 评分对 中文无影响（toLowerCase 对中文是 no-op）。如需严格大小写，需 plugin 扩展。
使用样本 · Usage
在阅读类课程的 Skim 步骤被频繁使用。"配对"是阅读策略中的基础工具，适合作为第一题建立信心。
