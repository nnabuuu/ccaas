是隐藏的理解度。

1

3

⋮⋮
4

⋮⋮
这是正确顺序的内在逻辑：先预测（建立预期），再略读（看结构），然后查读（找细节），最后评价（形成判断）。从抽象到具体，从输入到输出。

教师能观察到什么 From the Observation Panel

最终排序

每个学生提交的顺序

对错一目了然。错的学生具体把哪两项调换了也能直接看出来。

常见错位

全班最容易调换的两项

如果一半人把 "Scanning" 和 "Skimming" 互调，说明这两个概念学生混淆 —— 下一步该全班讲两者的区别。

尝试次数

一次过 vs 多次试错

一次排对 = 真正理解逻辑。试 3 次才对 = 在试错猜。老师能针对后者做后续提问。

完成时间

很快 vs 卡很久

20 秒排完 = 直觉对了或者瞎排；2 分钟才提交 = 在真的思考哪个先哪个后。看时间能补充正确率信息。

什么时候用 When to Reach for This

✓ 适合✗ 不适合 / 用别的
时间线（历史事件 / 故事情节 / 生物发育）顺序无所谓的题 —— 用 match 或 quiz
操作流程（化学实验步骤 / 数学解题步骤 / 算法 step）顺序有多个合理解 —— 用 discuss 让学生解释
程度排列（从轻到重 / 从近到远 / 从简到难）需要写理由 —— 用 image-upload 拍纸质排序
逻辑链（前提 → 推论 → 结论）多步独立计算 —— 用 guided-discovery
策略归纳（先做什么、再做什么）项目数太少（<3 项）—— 排序没有意义
05 —

出一道题需要准备什么 Authoring Checklist

出题清单

排序题最考验的是顺序背后的逻辑。如果"为什么这个在那个前面"你自己都说不清，学生只会觉得这题没道理。

1
先想清楚"正确顺序的内在逻辑"
是时间先后？因果链？抽象到具体？由轻到重？把这个逻辑写下来 —— 不能说清楚就说明题不该出。
2
列出 3-6 个待排项
少于 3 个排序没意义；多于 6 个学生会迷失。4 是甜点。每项最好一句话能说清楚，太长学生看不完。
3
检查：是否有多个合理顺序
如果"Scanning 在 Predicting 之前"也有道理 —— 这题就不适合做排序题，应该改成 discuss 让学生说理由。排序题要求唯一正确。
4
（推荐）准备一句"逻辑解释"
学生答对后展示。原则：不是"答案就是 1234"，而是解释为什么这是唯一正确顺序（"从抽象到具体，从输入到输出"）。让学生知道"下次遇到类似题怎么想"。
核心教学原则

顺序错了，
等于没真懂。

排序题最好用的地方在于：学生可能每个概念都认识，但如果排不出顺序，说明他没有真正建立起概念之间的因果 / 时序 / 层级关系。这是"碎片化知识"和"系统理解"的分水岭 —— 排序题精准地把它测出来。

06 —

常见疑问 FAQ

问题答案
排错可以重排吗？可以。无限重试。系统记录每次提交的快照。
必须全对才算对吗？是。顺序题没有"部分对" —— 错一对就是错。
可以做"分组+顺序"吗？不支持。如果是分组场景 → 用 matrix；如果组内还要排序 → 拆成多道题。
有时间限制吗？没有。学生可以反复调整，直到自己满意再提交。
能加图片吗？当前是纯文本卡片。如果需要"图片排序"（如漫画格子排序），等扩展版本。
能给每项加 hint 吗？当前版本不行（项本身就该是提示）。如果某项需要解释，把解释直接写到项文本里。

出题陷阱 · Pitfalls
