Select-evidence
点选证据题

在课文段落中点选 token作为证据 + 标注function（contrast / cause / example...）。客户端评分——sanitize 保留 correctFunction 等字段让前端实时反馈，无需后端往返。

查看源码 → type: "select-evidence"
01 —

Answer Key Schema

const SelectEvidenceSectionSchema = z.object({
  id:              z.string().min(1),
  label:           z.string().min(1),
  range:           z.array(z.number()).min(1),  // 段落号范围
  correctFunction: z.string().min(1),       // 该 section 的正确功能
  minHits:         z.number().int().min(1).optional(),
  hint:            z.string().optional(),
  aiCorrect:       z.string().optional(),
  aiPartial:       z.string().optional(),
});

const ParagraphTokenSchema = z.object({
  t:    z.string(),       // token 文本
  kind: z.string().optional(),  // 高亮类别
  why:  z.string().optional(),  // 该 token 的注释
});

const SelectEvidenceAnswerKeySchema = z.object({
  type:            z.literal('select-evidence'),
  functionOptions: z.array(z.string()).min(2),
  sections:        z.array(SelectEvidenceSectionSchema).nonempty(),
  paragraphTokens: z.record(z.string(), z.array(ParagraphTokenSchema)).optional(),
});
02 —

字段说明

字段类型用途
functionOptionsstring[]所有可选的功能标签（如 cause, effect, example, contrast）
sections[].id / .labelstringsection 标识 + 显示名
sections[].rangenumber[]该 section 涉及的段落号
sections[].correctFunctionstring该 section 的正确功能（必须在 functionOptions 中）
sections[].hintstring?答错提示
sections[].aiCorrect / .aiPartialstring?AI 反馈消息（答对/部分对时展示）
paragraphTokensRecord段落号 → token 数组，含 kind 用于前端高亮

04 —

Sanitize / Grade 行为

sanitize

剥离哪些字段

不剥离 correctFunction！因为客户端评分需要。也保留 hint / aiCorrect / aiPartial 和 paragraphTokens 的 kind/why。这是 故意的——见 schema 注释。

// 学生侧也能看到正确答案
{
  type: "select-evidence",
  functionOptions: [...],
  sections: [
    { id, label, range, correctFunction, ... }
  ],
  paragraphTokens: {...}
}
grade

评分规则（服务端备份）

服务端仍跑评分作为真实分数源（防止前端被改）：function 比较（case-insensitive）+ 可选 token hits 数。客户端 UI 实时反馈，但提交后服务端 score 是权威。

// 输出
{
  total: 75,
  byDimension: {
    "sec1": true,
    "sec2": false
  }
}
