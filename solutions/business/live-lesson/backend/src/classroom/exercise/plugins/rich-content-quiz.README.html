Rich-content-quiz
综合题

最复杂的 v1 题型之一。一道题分多个 parts，每个 part 独立 prompt + rubric + scaffold。Scaffold 在答错时渐进推送提示（思路 → 步骤 → 完整解答）。自管理提交流程 + 视觉评分。

01 —

Answer Key Schema

const ScaffoldStepSchema = z.object({
  title:  z.string().min(1),
  hintZh: z.string().optional(),
  widget: z.enum(['formula-animation', 'solution-display', 'procedure-steps']).optional(),
  props:  z.record(z.unknown()).optional(),
});

const ScaffoldLevelSchema = z.object({
  hintZh:    z.string().min(1),
  hintImage: z.string().optional(),
  steps:     z.array(ScaffoldStepSchema).optional(),  // 子步骤
});

const ScaffoldSchema = z.object({
  threshold: z.number().int().min(0),    // 答错几次开始 scaffold
  levels:    z.array(ScaffoldLevelSchema).nonempty(),  // 渐进级别
});

const RichContentPartSchema = z.object({
  id:             z.string().min(1),
  prompt:         z.string().min(1),
  expression:     z.string().optional(),
  rubric:         z.array(RubricItemSchema).nonempty(),
  sampleSolution: z.string().optional(),
  aiSystemPrompt: z.string().optional(),
  scaffold:       ScaffoldSchema.optional(),
  inputMethods:   z.array(...).optional(),
  accepts:        z.array(z.string()).optional(),
});

const RichContentQuizAnswerKeySchema = z.object({
  type:          z.literal('rich-content-quiz'),
  subType:       z.enum(['calculation']).optional(),
  parts:         z.array(RichContentPartSchema).optional(),
  rubric:        z.array(RubricItemSchema).optional(),
  aiSystemPrompt: z.string().optional(),
  // ... 与 image-upload 共享其他字段
}).refine(
  (ak) => (ak.parts && ak.parts.length > 0) || (ak.rubric && ak.rubric.length > 0),
  { message: 'rich-content-quiz: must have either parts or rubric' },
);
02 —

核心字段

字段类型用途
parts[]RichContentPart[]?多 part 模式：每个 part 独立 prompt + rubric + scaffold
parts[].scaffold.thresholdnumber答错几次后开始推送 scaffold（通常 1）
parts[].scaffold.levels[]ScaffoldLevel[]渐进提示级别：第 1 次错给 level[0]，第 2 次错给 level[1]，依此类推
parts[].scaffold.levels[].steps[]ScaffoldStep[]?该级别的子步骤（可含交互 widget）
parts[].scaffold.levels[].steps[].widgetenum?动画 widget：formula-animation / solution-display / procedure-steps
subType"calculation"?子类型，影响前端 UI（如显示公式动画）

04 —

Sanitize / Grade 行为

sanitize

剥离哪些字段

剥离 accepts、sampleSolution、aiSystemPrompt、rubric[].criteria。scaffold 字段保留为 hasScaffold: true 标志（学生不知道 hint 内容但前端可以触发"获取提示"按钮）。

// 学生看到
{
  type: "rich-content-quiz",
  parts: [
    { id, prompt, rubric (no criteria),
      hasScaffold: true }
  ]
}
grade

评分链路

每个 part 复用 ImageUploadGrader 流程（OCR + Vision）。并行评分（commit d3ded228），多 part 同时调 LLM 提速。Total = 各 part 加权平均。

// 输出（按 part）
{
  total: 80,
  partResults: {
    q1: { score: 100, feedback: "..." },
    q2: { score: 60,  feedback: "..." }
  }
}
05 —

什么时候用 Rich-content-quiz

适合场景不适合场景
多小题练习（计算 3 道 + AI 评分）单题（用 image-upload）
需要渐进 scaffold 的难题不需要 scaffold（用 image-upload 更轻）
分阶段提交（每 part 即时反馈）纯选择题（用 quiz）
迁移状态 · Status
尚未迁移。C 组最复杂题型之一。Stage 5 迁移时考虑与 image-upload 共享内部函数（OCR + vision grading 共用）。
特殊点 · Quirks
自管理提交（selfManagedSubmit=true），plugin 内部用 ctx.submit(step, data) 多次提交。Scaffold 在前端通过 onScaffoldPush 推送到侧边栏。
使用样本 · Usage
在 math-difference-of-squares 的"探索发现"步骤被使用（3 道乘法计算 + 渐进 scaffold）。是从具体到抽象教学的关键载体。
