Guided-discovery
引导式探究题

最复杂的题型之一。一道题分 4 类子步骤，渐进解锁 + 混合判分。布鲁姆分类学的工程化：从观察 → 应用 → 分析 → 综合。配合 OCR 缓存（commit d3ded228）避免重复视觉模型调用。

查看源码 → type: "guided-discovery"
01 —

4 类子步骤

子步类型认知层次（布鲁姆）核心交互
observation_choice记忆 / 理解看表格 / 高亮 → 单选判断（如"左边两个括号有什么关系？"）
formula_blanks应用公式模板填空（用 a、b 表示规律：(a+b)(a−b) = ?）
derivation_blank分析多行推导，每行可有填空（展开 (a+b)(a-b) = a²-ab+ab-b²）
text_blanks综合用文字描述规律（template + 多空 + swappable 顺序）
02 —

Answer Key Schema (Discriminated Union)

const GuidedDiscoveryStepSchema = z.discriminatedUnion('type', [
  GdObservationChoiceSchema,  // type: 'observation_choice'
  GdFormulaBlanksSchema,       // type: 'formula_blanks'
  GdDerivationBlankSchema,     // type: 'derivation_blank'
  GdTextBlanksSchema,          // type: 'text_blanks'
]);

const GuidedDiscoveryAnswerKeySchema = z.object({
  type:    z.literal('guided-discovery'),
  title:   z.string(),
  steps:   z.array(GuidedDiscoveryStepSchema).nonempty(),
  summary: z.object({
    formula:     z.string().optional(),
    name:        z.string().optional(),
    description: z.string().optional(),
  }).optional(),
});
03 —

核心字段（按子步类型）

子步.字段类型用途
observation_choice.table{expression, result}[]展示给学生看的表格（如 3 道乘法及结果）
observation_choice.highlights{same, opposite}高亮"相同项"vs"相反项"用不同颜色，引导观察
observation_choice.choices[]{id, options[2], correct}2 选 1 多个判断
formula_blanks.blanks[]{id, accepts, rejects, rejectHint}每空有 accepts 数组 + rejects 数组（典型错误）+ rejectHint（错答时显示）
derivation_blank.lines[]{text, blank?}多行推导，每行可含填空
derivation_blank.hintSteps[]{title, widget, props}scaffold 提示（widget = formula-animation / solution-display）
text_blanks.template / swappablestring / boolean文本模板 + 顺序可交换

示例 — 平方差公式 4 步

{
  "type": "guided-discovery",
  "title": "归纳平方差公式",
  "steps": [
    {
      "type": "observation_choice", "id": "observation_1",
      "title": "观察三道算式",
      "table": [
        { "expression": "(y+2)(y-2)", "result": "y²-4" },
        { "expression": "(3-a)(3+a)", "result": "9-a²" }
      ],
      "choices": [
        { "id": "c1", "options": ["完全相同","一和一差"], "correct": 1 }
      ]
    },
    {
      "type": "formula_blanks", "id": "formula_1",
      "title": "用字母表示",
      "blanks": [
        { "id": "b1", "label": "=",
          "accepts": ["a^2-b^2","a²-b²"],
          "rejects": ["a^2+b^2","ab"],
          "rejectHint": "注意运算符号：是减不是加" }
      ]
    },
    { "type": "derivation_blank", "id": "derivation_1", "title": "展开验证", "lines": [...] },
    { "type": "text_blanks", "id": "text_1", "title": "用文字描述", "template": "两个数的{{b1}}与{{b2}}的积..." }
  ],
  "summary": {
    "formula": "(a+b)(a-b) = a²-b²",
    "name": "平方差公式"
  }
}
05 —

Sanitize / Grade 行为

sanitize

剥离规则（per sub-step）

observation_choice 不剥离 correct（2 选 1，泄漏可忽略，换取客户端即时反馈）。其他子步剥离 accepts / rejects / rejectHint / blanks 答案。保留 hintSteps 教学脚手架。

// 学生看到的 formula_blanks
{
  type: "formula_blanks",
  blanks: [
    { id, label, placeholder }
    // accepts / rejects 被剥离
  ]
}
grade

混合评分链路

(1) 客户端判断 observation_choice / formula_blanks（accepts 匹配）。
(2) derivation_blank 手写走视觉模型（OCR 缓存 by image hash，相同图片复用结果）。
(3) text_blanks 走 LLM 同义判定。

// 输出（per sub-step）
{
  stepResults: {
    observation_1: { score: 100 },
    formula_1:     { score: 50,
                     rejectedBlanks: ["b1"] },
    derivation_1:  { score: 80, ocrCached: true }
  }
}

什么时候用 Guided-discovery

适合场景不适合场景
归纳公式（从具体到抽象）单步事实回忆（用 quiz）
多步骤渐进推理开放式表达（用 image-upload）
Productive Failure 教学场景简单结构化笔记（用 matrix）
需要混合多种评分方式不需要 OCR 的纯文本题（用 fill-blank）
迁移状态 · Status
尚未迁移。Stage 5 迁移最复杂——OCR 缓存逻辑 + 4 种子步骤 + 自管理提交。建议最后迁移，等其他类型 plugin 模式稳定。
特殊点 · Quirks
swappable：text_blanks 的多个空可交换顺序（{{b1}}{{b2}} 等价于 {{b2}}{{b1}}）。rejectHint 是典型错误的领域编码——这是 GD 区别于其他题型的核心。
使用样本 · Usage
