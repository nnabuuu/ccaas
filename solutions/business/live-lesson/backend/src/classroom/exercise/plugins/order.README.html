Order
序列排序题

给一组打乱的项目，学生拖拽排成正确顺序。0/100 二元评分——全对得满分，错任意一项 0 分。校验严格：item 数量、索引唯一性、范围都由 Zod refine 验证。

查看源码 → type: "order"
01 —

Answer Key Schema Zod

服务端结构（含答案） 
const OrderAnswerKeySchema = z.object({
  type:         z.literal('order'),
  items:        z.array(z.string()).min(2),
  correctOrder: z.array(z.number().int().nonnegative()),
})
.refine(  // length 必须相等
  (ak) => ak.correctOrder.length === ak.items.length,
  { message: 'order: correctOrder.length must equal items.length' },
)
.refine(  // 所有索引必须在范围内
  (ak) => ak.correctOrder.every((v) => v < ak.items.length),
  { message: 'order: correctOrder values must be < items.length' },
)
.refine(  // 不允许重复索引（必须是排列）
  (ak) => new Set(ak.correctOrder).size === ak.items.length,
  { message: 'order: correctOrder must contain each index exactly once' },
);

示例 Answer Key Example

{
  "type": "order",
  "items": [
    "Venus of Hohle Fels (35,000 years ago)",
    "Egyptian paintings of slim women",
    "African fattening room tradition",
    "Modern media slim ideal"
  ],
  // 时间顺序: 0 (最古老) → 1 → 2 → 3 (最现代)
  "correctOrder": [0, 1, 2, 3]
}

// 学生提交
// data.order: ["Modern media...", "Egyptian...", "African...", "Venus..."]
// 或 numeric: data.order: [3, 1, 2, 0]
04 —

Sanitize / Grade 行为 Plugin Methods

sanitize

剥离哪些字段

从 answerKey 生成 ExerciseSpec，剥离 correctOrder。保留 items 数组（学生可见，前端通常会随机打乱顺序显示）。

// sanitize 后学生看到的
{
  type: "order",
  items: ["Venus...", "Egyptian...",
          "African...", "Modern..."]
}
grade

评分规则

所有位置都对才得分。0/100 二元评分。支持 student 提交字符串数组、数字索引数组，或 { label } 对象数组。比较 case-insensitive。

// 输出
{
  total: 100,
  byDimension: { correct: true }
}
// 或
{
  total: 0,
  byDimension: { correct: false }
}
05 —

什么时候用 Order When to Use

适合场景不适合场景
时间排序（历史事件、生命周期、流程步骤） 需要部分得分（用 quiz 或 matrix）
逻辑推理顺序（论证结构、解题步骤） 不存在唯一正确顺序的题目（用 stance）
大小排序（数据、长度、规模） 项目数量大（>8）— 拖拽体验差
因果链（A 导致 B 导致 C） 非线性的关联（用 map）
06 —

校验细节 Validation Quirks

错误例子原因
length mismatch items 4 个，correctOrder 3 个 排序题必须给出完整排列，长度必须相等
out of range items 4 个，correctOrder 含 [0, 1, 4] 索引必须 < items.length
duplicate index correctOrder: [0, 1, 1, 2] 必须是排列，每个索引出现且仅出现 1 次
