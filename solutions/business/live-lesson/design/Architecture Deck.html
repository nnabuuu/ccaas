编辑器 UI据 schema 渲染参数表单
Agent据 schema 生成合法配置
校验层据 schema 拦截非法状态
模块 Ref 机制
模块定义是独立 JSON 文件，可被 $ref 引用到执行 manifest 中。编辑模块定义文件，所有引用处同步更新。
Schema 约束
模块类型由后端提供，只读、不可扩展。教师和学校不能自定义新的模块类型。
模块 Schema
每个模块的参数分为两类字段
结构化配置
字段类型UI 渲染 durationnumber滑块 minTurnsnumber数字输入 difficultyenum下拉选择 passScorenumber滑块
Declarative Prompt

⚠ 两组字段读取者不同、判断标准不同。混合定义会导致调整 tutor 行为时意外改变通过条件。

"引导学生理解美的实践是一种文化语言。鼓励从课文中找证据，使用苏格拉底式追问，不直接给出答案。"
→ AI Evaluator
Completion Rubric
定义学生达标判断标准
"学生能说出美的实践是表达身份认同和文化归属的方式，需要引用至少一个课文中的具体实践作为证据。"

Coverage 覆盖检查
执行设计中的模块是否覆盖了教案中定义的所有教学要求。
Conflict 冲突检测
执行设计与教案之间是否存在矛盾或不一致。
Lint 遗漏与冗余
教案有但执行没承接的遗漏，或执行有但教案未定义的冗余。
设计选择: 教案与执行设计之间无 foreign key。一致性是 soft constraint，由 AI lint 保证。教师修改教案措辞不会导致执行设计侧的引用失效。
总结
四个架构原则
文件即真相
所有数据最终落到文件系统。Agent 和教师编辑共享同一套存储。没有隐藏的黑盒数据库。
Single source of truth
Soft Constraint
教案与执行设计之间无 FK。一致性由 AI lint 保证，避免硬关联的级联失效。
AI lint, not foreign key
Schema 驱动
模块类型由后端 schema 定义，只读不可扩展。同一份 schema 服务 UI、Agent、校验三方。
One schema, three consumers
原样消费
Manifest 是 design-time 和 runtime 的唯一合同。学生端直接消费 JSON，不编译不转换。
No compilation step
