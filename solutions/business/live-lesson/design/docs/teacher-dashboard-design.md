# 课堂观察台（教师端）设计文档

> 版本：v2 · 2026-04-21
> 文件：`surfaces/teacher-v2.html`

---

## 一、产品定位

课堂观察台是教师在 AI 一对一自主推进课堂中的**实时监控面板**。42 名学生各自以不同速度推进 5 个 Step，教师需要在不打断学生的前提下，快速识别需要介入的情况。

### 核心用户场景
| 场景 | 频率 | 紧迫度 |
|------|------|--------|
| 扫一眼全班进度分布 | 每 2-3 分钟 | 低 |
| 发现哪个 Step 是瓶颈 | 每 5 分钟 | 中 |
| 识别具体卡住的学生 | 实时 | 高 |
| 查看某学生的详细作答和 AI 对话 | 按需 | 高 |
| 回溯某学生的历史表现 | 按需 | 低 |

### 设计原则
1. **Glanceable** — 关键信息在视线停留 2 秒内可获取
2. **Progressive Disclosure** — 概览→卡片→弹窗，三层递进
3. **Attention-first** — 需要介入的信息自动浮出，不需要老师逐个检查

## 三、设计系统

### 色彩语义

| 语义 | 变量 | 色值 | 使用场景 |
|------|------|------|----------|
| 正常/完成 | `--green` / `--green-dot` | #2d6612 / #5a9a3a | 已完成状态、正确率高 |
| 进行中 | `--blue` | #1a5fa0 | 当前活跃的步骤或学生 |
| 警告/卡住 | `--amber` / `--amber-dot` | #7a4d0e / #c48a1e | 卡住学生、正确率偏低 |
| 错误 | `--red` | #942929 | 错误答案 |
| AI 相关 | `--ai` / `--ai-dot` | #4a4387 / #7a72b5 | AI 对话触发 |
| 阅读/讲授 | `--lecture` | #6b5b3a | Listen 阶段 |
| 中性 | `--t1/t2/t3` | #1c1c1a / #5c5b56 / #9c9a92 | 文字层级 |

### 排版
- 字体：Plus Jakarta Sans（主）+ PingFang SC（中文回退）
- 数字：`font-variant-numeric: tabular-nums`（对齐）
- 层级：标题 13px/700，正文 11px/500，标签 9px/600 大写

### 圆角体系
- 卡片：10px
- 弹窗：14px
- 小标签/badge：3-5px
- 头像/圆形指示器：50%

## 四、组件设计

### 4.1 Health Cards（健康指标）

**目的**：一行 4 格，回答"全班现在怎么样"。

| 指标 | 内容 | 视觉变体 |
|------|------|----------|
| 最快进度 | 最前面的 Step 编号 + 人数 | `.good` 绿色背景 |
| 中位进度 | 大多数学生所在 Step | 默认中性 |
| 卡点学生 | 卡住人数 + 集中位置 | `.warn` 琥珀色背景 |
| AI 对话 | 总对话轮数 + 触发人数 | 默认中性 |

**设计决策**：只用 `.good` 和 `.warn` 两种语义色，不用 `.bad`——避免教师焦虑，卡住学生是正常教学现象。

### 4.2 Step Cards（课堂进程卡片）

**目的**：取代原始的泳道行。每张卡片是一个 Step 的完整信息单元。

**信息层次**（从上到下）：
1. **头部**：Step 编号 + 名称 + 题型描述 + 右侧 badges
2. **指标条**：正确率 + 最弱维度 mini bar + AI 触发人数
3. **学生点阵**：按状态着色的姓名块（done/prog/stuck/reading）

**Badges 设计**：
| Badge | 背景色 | 触发条件 |
|-------|--------|----------|
| `N 人` | 灰色/蓝色(≥20人) | 始终显示 |
| `AI N轮` | 紫色/红色(≥20轮) | 始终显示 |
| 告警标签 | 琥珀色 | 有 `alertTag` 或 ≥5 人卡住 |

**交互**：
- Hover：边框加深 + 轻微投影
- Click：打开 Step 详情弹窗（而非内联展开，避免挤压主布局）
- 学生姓名块可直接点击 → 打开学生详情弹窗

**设计决策**：
- 为什么不内联展开？信息量太大（维度条形图 + 问题列表 + 学生名单），内联会破坏整体节奏。弹窗提供充足空间。
- 为什么把 AI 数据嵌入卡片？原设计中 AI 助教活动是独立 section，老师无法将 AI 数据与 Step 关联。合并后，每个 Step 的 AI 热度一目了然。

### 4.3 Step 详情弹窗

**目的**：点击 Step 卡片后展开的详细视图。

**布局**：
```
┌─────────────────────────────────────────┐
│ Header: ⓷ Scan & Build  15min·信息矩阵  ✕ │
├─────────────────────────────────────────┤
│ Stats: 当前人数 | 中位用时 | AI轮次 | AI人数 │
├─────────────────────────────────────────┤
│ 各维度正确率（条形图）                     │
│  Where  ████████████████████░░  92%      │
│  What   █████████████░░░░░░░░  67%      │
│  Why    ████████░░░░░░░░░░░░░  42%      │
├─────────────────────────────────────────┤
│ 主要问题                                  │
│  ⚠ 7 人将 Myanmar 与 Indonesia 合并       │
│  ⚠ Why 列空缺率最高 (42%)                │
├─────────────────────────────────────────┤
│ 当前在此步的学生 [可点击跳转]              │
│  [陈昕妍] [李奕辰] [王译文🟣] ...          │
└─────────────────────────────────────────┘
```

**交互**：学生姓名块可点击 → 关闭当前弹窗 → 打开学生详情弹窗。

### 4.4 学生详情弹窗

**目的**：查看单个学生的全步骤历史和当前作答情况。

**核心设计理念**：教师需要回答三个问题：
1. 这个学生走到哪了？（进度）
2. 整体表现如何？（跨 Step 的模式）
3. 哪里需要帮助？（注意力信号）

**布局**：
```
┌──────────────────────────────────────────────┐
│ Header: [王] 王译文  当前在 Step 3             ✕ │
├──────────────────────────────────────────────┤
│ Journey Strip（旅程条）                        │
│ ┌──┐   ┌──┐   ┌════┐   ┌┄┄┐   ┌┄┄┐         │
│ │①│───│②│───║ ③ ║╌╌╌│④│╌╌╌│⑤│         │
│ │✓ │   │△ │   │● 进│   │  │   │  │         │
│ │3:20│ │7:10│ │8:12│   │未到│   │未到│         │
│ └──┘   └──┘   └════┘   └┄┄┘   └┄┄┘         │
│        ⚠需关注           （灰色虚线=未到达）      │
├─────────────────────┬────────────────────────┤
│ 左栏：作答详情        │ 右栏：AI对话 或 班级对比 │
│                     │                        │
│ 学生矩阵 / 答题记录   │ 有 AI 对话 → 聊天记录   │
│                     │ 无 AI 对话 → 班级对比卡  │
│                     │  用时: ██████▎ 7:10     │
│                     │  正确: ████▎   75%      │
│                     │  AI轮: █▎     1        │
│                     │  ─ 该学生  ─ 班级  │中位  │
└─────────────────────┴────────────────────────┘
```

**Journey Strip 设计**：
- 替代原来的 Tab 导航，同时展示全部 5 个 Step 的状态
- 使用三通道编码（图标 + 文字 + 颜色）消除歧义：

| 状态 | 图标 | 文字 | 颜色 |
|------|------|------|------|
| 完成且正确 | ✓ | 已完成 | green |
| 完成但部分正确 | △ | 部分正确 | amber |
| 进行中 | ● | 进行中 | blue |
| 卡住 | ⚠ | 需关注 | amber |
| 未到达 | ○ | 未到达 | grey + 虚线边框 |

- 需要关注的 Step 节点：amber 边框 + 脉冲动画的"⚠ 需关注"标签
- 当前选中的 Step：蓝色边框 + 蓝色背景
- 点击任一节点 → 下方切换对应 Step 的详情

**右栏空间利用策略**：
- 有 AI 对话 → 显示完整聊天记录
- 无 AI 对话 → 显示"班级对比卡"：该学生 vs 班级中位数的用时/正确率/AI轮次对比条形图。这为教师提供了**上下文**——不仅知道学生做得好不好，还知道相对于班级如何。

**固定尺寸**：弹窗高度固定 680px，切换 tab 时不跳动，减少认知负担。

## 五、右栏：问题聚类

**目的**：将学生的高频问题按 Step 分组聚合，帮助教师判断是否需要全班讲解。

**设计**：
- 按 Step 分组，sticky header
- 每条问题显示出现次数（加粗数字）+ 问题文本
- 高频问题（≥4 次）：数字变琥珀色 + "高频" 标签
- 上升趋势问题：显示 ↑N 标记

**设计决策**：右栏只保留问题聚类，移除原来的 AI 助教活动和 Step 质量面板（已合并到左侧 Step 卡片）。这让右栏职责单一：**"学生在问什么"**。

## 六、时间轴

**目的**：允许教师回溯任意时刻的课堂状态。

- 拖拽滑块或点击跳转
- 实时模式：绿色标签 "实时 · 18:22"
- 回放模式：琥珀色标签 "回放 · 12:30" + 显示"直播"按钮
- Task 边界标记：轨道上的竖线

---

## 七、观察要点 + 教学参考

**观察要点**（2×2 网格）：AI 自动生成的教学洞察卡片。
- 告警类（琥珀色左标记）：T3 瓶颈、Listen 跳过
- 信息类（蓝色左标记）：进度分散度、AI 使用分布

**教学参考**（折叠面板）：低优先级的教学建议，默认折叠，避免干扰。

---

## 八、交互流

### 主要路径

```
全班概览（主页面）
  │
  ├─ 点击 Step 卡片 → Step 详情弹窗
  │    └─ 点击学生姓名 → 学生详情弹窗
  │
  ├─ 点击学生姓名块 → 学生详情弹窗
  │    └─ 点击 Journey Strip 节点 → 切换 Step 详情
  │
  └─ 拖动时间轴 → 回溯历史状态
```

### 弹窗层级
- Step 详情弹窗：z-index 90
- 学生详情弹窗：z-index 100（可覆盖 Step 弹窗）
- 从 Step 弹窗点击学生 → 先关闭 Step 弹窗，再打开学生弹窗
