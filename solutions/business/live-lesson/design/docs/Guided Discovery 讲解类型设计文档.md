# Guided Discovery 讲解类型 · 设计文档

> 新增 Guided Discovery（引导探究）题型：逐步引导学生发现数学规律  
> 示范任务：平方差公式 $(a+b)(a-b) = a^2 - b^2$

---

## 1. 概述

Guided Discovery（引导探究 / 讲解）是一种新的课堂执行题型，专为**概念推导和公式发现**设计。教师不直接讲授结论，而是通过一系列递进式问题引导学生自主观察、归纳、验证、表述，最终"发现"公式或规律。

与现有的 Practice（练习）、Matching Card（结构解码）、Matrix（矩阵构建）等题型并列，覆盖数学定理推导、物理规律归纳等概念教学场景。

### 核心特征

| 维度 | 设计选择 |
|---|---|
| **交互模式** | 逐步渐进：完成当前问后才显示下一问 |
| **题目类型** | 混合型 — 观察表格 + 二选一 + 填空（键盘/手写/拍照）+ 文字填空 |
| **AI 介入模式** | 即时反馈：每个填空/选择提交后立即给出正误判断和引导 |
| **进度展示** | Sticky 进度条，始终可见，标注步骤名称 |
| **最终产出** | 学生自己"写出"公式 + 文字描述，形成完整认知闭环 |

### 设计原则

沿用平台四项产品哲学，增加引导探究特有原则：

- **学生是发现者，不是接收者** — 每一步都由学生主动回答，AI 只在回答后给反馈
- **渐进式揭示** — 问题按认知逻辑排列（观察 → 符号化 → 验证 → 语言化），前一步是后一步的基础
- **即时反馈，允许重试** — 选错/填错不锁死，给出提示后可重新选择/修改
- **多模态输入** — 数学公式填空支持键盘、手写、拍照三种输入方式
- **进度始终可见** — Sticky 进度条让学生随时知道自己在哪一步、还有几步

### 2.2 进度条（Step Progress）

| 属性 | 设计 |
|---|---|
| **位置** | Sticky 固定在内容区顶部，滚动时始终可见 |
| **布局** | 全宽水平排列，4 个步骤等分，步骤间用连接线填充 |
| **步骤标识** | 编号圆圈 (26px) + 下方步骤名称 |
| **状态** | 未到达 = 灰色边框 · 当前 = teal 高亮 · 已完成 = green + ✓ |
| **连接线** | 未填充 = `--border` · 已填充 = `--green` |

四个步骤名称：**观察规律** → **符号表示** → **验证公式** → **文字描述**

### 2.3 问题卡片（Question Card）

每个问题为一个独立卡片（`.q-card`），结构：

| 组件 | 说明 |
|---|---|
| **Header** | teal 编号圆圈 + 问题标题 |
| **Body** | 问题内容、交互区域、AI 反馈 |
| **渐进动画** | `cardIn` 0.35s ease，从下方淡入 |
| **锁定状态** | 未到达的问题 `opacity: .4; pointer-events: none` |

卡片类型：

| 问题 | 卡片类型 | 交互元素 |
|---|---|---|
| Q1 观察规律 | 表格 + 多个二选一 | `InlineChoice` × 4 |
| Q2 符号表示 | 居中等式填空 | `MathInput` × 2 |
| Q3 验证公式 | 推导过程 + 末行填空 | `MathInput` × 1 |
| Q4 文字描述 | 文字填空 | `<input>` × 2（简单文字） |

### 2.4 交互组件

#### InlineChoice（行内二选一）

嵌入在文本段落中的选择按钮，用于 Q1 的"相同项/相反项"选择。

| 状态 | 视觉表现 |
|---|---|
| **未选择** | teal 1.5px 边框 + 呼吸动画，右侧显示 teal「选一个」标签 |
| **选对** | green 边框 + green 背景，锁定不可修改 |
| **选错** | 选中项变红，可直接点另一个重选，右侧显示 amber「再选 ↻」标签 |

设计要点：
- 选错**不锁死** — 学生可以直接点击另一个选项重新选择
- 只有选对才锁定，避免挫败感
- 「选一个」提示明确引导学生这里需要交互

#### MathInput（多模式输入）

可点击展开的填空组件，支持三种输入方式。用于 Q2、Q3 的数学公式填空。

**折叠态：**

| 状态 | 视觉表现 |
|---|---|
| **空白** | teal 虚线边框，显示引导文字（如"等号左边"） |
| **已填入** | 实线边框，显示内容预览（文字/画布缩略图/照片缩略图） |
| **hover 已填入** | 显示「点击修改」灰色提示 |
| **正确** | green 边框 + green 背景 |
| **错误** | red 边框 + red 背景 |

**展开态：**

```
┌─ Tab Bar ─────────────────────────────────┐
│ [🔤 键盘] [✏️ 手写] [📷 拍照]    [↑ 收起] │
├───────────────────────────────────────────┤
│                                           │
│ (根据当前 tab 显示对应输入区域)              │
│                                           │
├───────────────────────────────────────────┤
│                           [✓ 确认输入]     │
└───────────────────────────────────────────┘
```

三种模式：

| 模式 | 内容 |
|---|---|
| **键盘** | 文本输入框 + 输入提示「² 可输入 ^2」。支持 Enter 确认 |
| **手写** | 与 Practice v3 一致的画布（1200×480、方格纸、笔/橡皮/撤销/清除工具栏）。单页 |
| **拍照** | 拖拽/点击上传区域，上传后显示预览 + 重新选择按钮 |

**确认输入按钮：** 面板底部右侧，teal 色，有内容时才显示。点击后：
1. 生成画布缩略图（手写模式）
2. 收起面板
3. 折叠态显示内容预览

#### AI 反馈（Inline Feedback）

每个问题提交后即时显示的反馈卡片：

| 类型 | 视觉 | 图标 |
|---|---|---|
| **正确** | green 背景 + green 图标 | ✓ |
| **错误** | red 背景 + red 图标 | ✗ |

反馈内容结构：`<strong>判定词</strong> + 解释/引导`

错误反馈附带「修改答案」按钮，点击后清除提交状态，可重新作答。

### 2.5 公式总结卡（Summary Card）

Q4 完成后出现，teal 背景，居中显示：
- 上方小标签：`公式总结`
- 中间 KaTeX display 模式：$(a+b)(a-b) = a^2 - b^2$
- 下方标题：**平方差公式**
- 文字描述

## 3. 题目流程（逐步渐进）

```
    ┌──────────┐
    │ Q1 观察   │ ← 4 个 InlineChoice，全部选对后
    └────┬─────┘
         │ [继续下一问 →]
         ▼
    ┌──────────┐
    │ Q2 符号化  │ ← 2 个 MathInput，提交正确后
    └────┬─────┘
         │ [继续下一问 →]
         ▼
    ┌──────────┐
    │ Q3 验证   │ ← 1 个 MathInput，提交正确后
    └────┬─────┘
         │ [继续下一问 →]
         ▼
    ┌──────────┐
    │ Q4 语言化  │ ← 2 个文字填空，提交正确后
    └────┬─────┘
         │ [查看总结 →]
         ▼
    ┌──────────┐
    │ 公式总结卡  │
    └──────────┘
```

每步完成后：
1. 显示 AI 正确反馈
2. 出现「继续下一问 →」按钮
3. 点击后，下一个问题卡片带动画出现
4. 进度条同步更新
5. 自动滚动到新卡片

### 3.2 步骤进度漏斗（Step Funnel）

引导探究的核心观察视图。教师一眼看出全班在 4 步中的分布：

| 要素 | 设计 |
|---|---|
| **行数** | 4 行，对应 4 个步骤 |
| **条形颜色** | green = 已通过该步，blue = 当前正在该步 |
| **数值** | 右侧显示「已通过数」+ 「+当前作答数」 |
| **基底** | `--surface2` 灰色背景，总长 = 全班人数 |
| **图例** | 底部：■已通过 ■当前作答 |

漏斗效果：步骤越靠后，绿色条越短 → 教师立即知道哪步是瓶颈。

### 3.3 各步骤详情卡（Step Detail Cards）

2×2 网格，每张卡片对应一个步骤：

| 信息 | 展示方式 |
|---|---|
| **步骤编号 + 名称** | teal 圆圈 + 标题 |
| **通过人数** | green badge |
| **当前作答人数** | blue badge（有人时显示） |
| **重试人数** | amber badge + "X人多次尝试" |
| **错误聚类** | 红色 badge + 错误描述 + 涉及学生姓名 |

错误按描述自动聚类（相同描述的学生合并显示）。

### 3.4 学生列表（右栏）

| 元素 | 说明 |
|---|---|
| **状态圆点** | done(灰) / prog(黑) / idle(空心) |
| **姓名** | 未开始的学生灰色 |
| **步骤标签** | 进行中的学生显示 blue「第X步」badge |
| **完成标记** | 已完成全部显示 green ✓ |
| **错误标记** | 有错误记录的学生显示 red ● |
| **筛选** | 全部 / 已完成 / 有错误 / 进行中 |

点击任意学生 → 打开详情弹窗。

## 4. 数据模型

### 4.1 题目定义

```javascript
{
  type: 'guided_discovery',
  title: '平方差公式',
  steps: [
    {
      id: 'observe',
      title: '观察规律',
      type: 'observation_choice',
      table: [
        { expression: '(x+2)(x-2)', result: 'x^2-4' },
        { expression: '(3+y)(3-y)', result: '9-y^2' },
        { expression: '(2a+b)(2a-b)', result: '4a^2-b^2' }
      ],
      highlights: {
        same: { color: 'red', terms: [['x','x'], ['3','3'], ['2a','2a']] },
        opposite: { color: 'blue', terms: [['+2','-2'], ['+y','-y'], ['+b','-b']] }
      },
      choices: [
        { id: 'c1a', prompt: '完全相同的项我们称为', options: ['相同项','相反项'], correct: 0 },
        { id: 'c1b', prompt: '只有符号相反的项叫做', options: ['相同项','相反项'], correct: 1 },
        { id: 'c2a', prompt: '结果可看成...的平方', options: ['相同项','相反项'], correct: 0 },
        { id: 'c2b', prompt: '减...的平方', options: ['相同项','相反项'], correct: 1 }
      ]
    },
    {
      id: 'symbolize',
      title: '符号表示',
      type: 'formula_blanks',
      prompt: '用 a, b 表示规律',
```

## 6. 与 Practice 题型的对比

| 维度 | Practice（练习） | Guided Discovery（引导探究） |
|---|---|---|
| **目标** | 检验计算能力 | 引导发现规律/公式 |
| **结构** | 单题 + 渐进提示 | 多步递进问题链 |
| **输入方式** | 统一手写/拍照多页 | 混合：选择 + 填空 + 手写 |
| **AI 介入** | 提交后批改，最多 2 次 | 每步即时反馈，可重试 |
| **进度** | 单题无进度 | Sticky 多步进度条 |
| **面板** | 右侧提示面板 | 无独立面板，反馈内嵌 |
| **最终产出** | 计算结果 | 学生自己写出的公式 + 文字 |

## 8. 设计决策记录

### 8.1 为什么用 InlineChoice 而不是标准按钮？

**问题**：Q1 的选择嵌入在一段连贯的文字描述中（"这些完全相同的项我们称为 ___"），如果用独立的按钮组件会打断阅读节奏。

**选择**：InlineChoice 嵌入在文本流中，像完形填空的空格。理由：
- 保持文本连贯性，学生在"读"的过程中自然"填"
- 视觉上用 teal 边框 + 呼吸动画吸引注意
- 「选一个」标签解决引导性不足的问题

### 8.2 为什么选错不锁死？

**初版**：选错后锁定，显示正确答案，需要点击"重试"按钮。  
**问题**：增加了操作步骤，打断思考流程。

**最终选择**：选错后按钮变红，但可以直接点另一个选项重新选。理由：
- 减少操作步骤
- 鼓励立即修正，而非等待重置
- amber「再选 ↻」标签温和引导

### 8.3 为什么 MathInput 需要"确认输入"按钮？

**初版**：用户手写/拍照后只能点"收起"关闭面板。  
**问题**：用户不确定内容是否已"填入"到等式中，缺乏确定感。

**最终选择**：面板底部显示 teal 色「✓ 确认输入」按钮（有内容时才出现）。理由：
- 明确的"提交"语义，学生知道内容已填入
- 关闭面板后显示内容预览（文字/缩略图），形成闭环
- 键盘模式额外支持 Enter 确认

### 8.4 为什么 Q4 不用 MathInput？

Q4 的填空只需要填一个汉字（"和" / "差"），用完整的多模式输入组件过于复杂。简单的 `<input>` 底线样式即可，降低认知负担。

### 8.5 为什么进度条要 sticky？

**初版**：进度条跟随内容滚动，到后面的问题时看不到。  
**问题**：学生不知道自己还有几步要完成。

**最终选择**：Sticky 固定在内容区顶部。理由：
- 随时可见当前进度
- 已完成的步骤变绿色 ✓，给予成就感
- 不占用太多空间（高度约 60px）

### 8.6 为什么教师端用漏斗而不是复用进度条？

**问题**：学生端的 4 步进度条是个人视角（一个人在哪步）。教师端需要**全班视角**（每步有多少人通过）。

**选择**：4 行水平条形图（漏斗形），而非圆形进度条或数字表格。理由：
- 漏斗天然传达"越往后人越少"的认知——教师瞬间看出瓶颈在哪步
- 绿色（已通过）+ 蓝色（当前作答）双色，区分"安全"和"需要等待"的学生
- 与 Practice 的提示使用分布条形图视觉一致，教师不需要学习新模式

### 8.7 为什么教师端详情卡用 2×2 网格？

**备选**：垂直列表（每步一行）或 Tab 切换。

**选择**：2×2 网格。理由：
- 4 个步骤刚好铺满，不浪费空间
- 教师可同时对比各步骤的错误分布，而非逐个切换
- 卡片内信息密度适中：通过数 + 重试数 + 错误聚类，不超载

## 9. 可扩展方向

- **更多题目类型** — 支持拖拽排序、连线匹配等交互形式
- **分组协作** — 不同学生/小组做不同的观察任务，汇总后全班讨论
- **AI 自适应反馈** — 接入 AI Skill API，根据学生的具体错误给出个性化提示
- **模板化** — 将 Guided Discovery 抽象为可配置的 JSON 模板，教师可自定义问题链
- **回溯查看** — 允许学生点击进度条回看已完成的步骤（当前只能向前）
- **教师端实时推送** — 教师可针对某一步骤向全班或指定学生推送额外提示
- **步骤耗时分析** — 教师端增加每步平均耗时统计，辅助判断哪步需要更多讲解时间
- **历史对比** — 同一引导探究在不同班级的完成情况对比，辅助教研
