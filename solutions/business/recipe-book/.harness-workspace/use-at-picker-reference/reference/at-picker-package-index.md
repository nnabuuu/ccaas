## 核心设计决策总结

### 决策 1: 双触发模型
`@` 引用实体，`/` 插入命令。**两者都只是编辑消息，发送时才执行。**

### 决策 2: 双焦点模型（fzf 模式）
搜索框和列表是两个焦点区域，`↓` 是桥梁。**按键语义完全由焦点位置决定，不依赖隐藏状态。**

| 按键 | 搜索框 | 列表 |
|---|---|---|
| Space | 空格 | Toggle 添加 |
| Enter | 选第一个结果 | 确认当前项 |
| ← → | 光标移动 | 导航层级 |

### 决策 3: EntityRef 路径寻址
每个 Token 携带完整路径（`[category, entity, child]`），路径即语义。同一实体从不同入口产生不同 Token。

### 决策 4: Ref 无深度限制 + Loop 检测
引用链可以无限深入，但检测循环（navStack 中已存在的 ID → ⟲ 标记 + shake 反馈）。

### 决策 5: Enter 确认 / Esc 退出
Enter = 正向确认，行为只取决于高亮项类型（分类→进入，实体→选择）。
Esc = 退一步，永不丢弃已选内容。

### 决策 6: 微交互动画
每个操作都有可感知的视觉反馈：添加→闪绿+弹入，Loop 阻断→shake+tooltip，层级切换→淡入。
