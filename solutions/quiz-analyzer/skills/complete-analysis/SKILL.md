# Skill: 完整题目分析

## 目标

为题目生成完整的 10 个维度的分析，包括：
1. 整体分析 (quizAnalysis)
2. 知识点标签 (knowledgePointTags)
3. 解题思路 (thinkingProcess)
4. 解题步骤 (solutionSteps)
5. 常见错误 (commonMistakes)
6. 知识缺口分析 (knowledgeGapAnalysis)
7. 难度等级 (difficulty)
8. 难度说明 (difficultyRationale)
9. 时间预估 (timeEstimate)
10. 相关题目推荐 (relatedQuizzes)

## 触发条件

**关键词触发**：
- "完整分析" (优先级 10)
- "生成分析报告" (优先级 10)
- "全面分析这道题" (优先级 10)
- "分析题目" (优先级 9)
- "详细分析" (优先级 9)

**使用场景**：
1. 教师需要快速生成题目的完整分析报告
2. 题库维护人员批量生成题目分析
3. 学生想了解题目的详细解题思路和知识点

## 标准工作流

### 步骤 1: 获取题目信息

使用 `get_quiz_details` 工具获取题目完整信息：

```json
{
  "quizId": "<题目 ID>"
}
```

**返回信息**：
- 题目内容 (content)
- 科目信息 (subject)
- 年级 (gradeLevel)
- 题型 (quizType)
- 已有知识点（如果存在）
- 已有分析（如果存在）

**错误处理**：
- 如果题目不存在，提示用户检查题目 ID
- 建议使用搜索功能查找题目

---

### 步骤 2: 知识点分析

#### 2.1 分析题干知识点

**目的**：识别题目本身涉及的知识点

**方法**：
1. **提取关键词**
   - 从题干中提取数学/物理/化学等学科术语
   - 识别题目类型相关的关键词
   - 例如："一元二次方程"、"因式分解"、"平方差公式"

2. **搜索候选知识点**
   - 使用 `search_knowledge_points` 工具搜索每个关键词
   - 参数：`{ query: "关键词", subjectId: "科目ID", limit: 5 }`
   - 收集所有候选节点

3. **展开有子节点的候选项**
   - 检查候选节点的 `children_count`
   - 如果 `children_count > 0`，使用 `get_children_nodes` 展开
   - 将子节点也加入候选池

4. **从候选池中选择相关知识点**
   - 基于题干内容判断相关性
   - 标记 `source: 'question'`
   - 设置置信度（0.0-1.0）
   - 避免遗漏综合题的多个知识点

#### 2.2 分析解答知识点

**目的**：识别解答过程中涉及的知识点

**方法**：
1. 分析标准答案或解答过程
2. 识别解答中使用的公式、定理、方法
3. 搜索对应的知识点
4. 标记 `source: 'solution'`

#### 2.3 合并知识点

**规则**：
- 题干和解答都涉及的知识点 → `source: 'both'`
- 设置置信度（0.0-1.0）：
  - 直接明确涉及 → 0.90-1.0
  - 间接相关 → 0.70-0.89
  - 可能相关 → 0.50-0.69
- 生成知识点路径（使用 `get_node_path`）

**输出格式**（knowledgePointTags）：
```typescript
[
  {
    id: "kp-001",
    name: "一元二次方程",
    confidence: 0.95,
    verified: false,
    level: 2,
    path: ["数学", "代数", "方程"],
    source: "both",
    note: null
  },
  {
    id: "kp-045",
    name: "判别式",
    confidence: 0.88,
    verified: false,
    level: 3,
    path: ["数学", "代数", "方程", "一元二次方程"],
    source: "solution",
    note: "解答过程中使用了判别式判断解的情况"
  }
]
```

### 步骤 3: 生成整体分析 (quizAnalysis)

生成 Markdown 格式的整体分析：

**模板**：
```markdown
# 题目特点

[1-2 句话概括题目类型和特点，例如："这是一道考察一元二次方程的解答题，需要综合运用因式分解和判别式知识。"]

# 考点分析

- **主要考点**：[知识点 1]、[知识点 2]
- **综合性**：[是否涉及多个知识点的综合应用，如何综合]
- **难点**：[题目的主要难点在哪里，为什么难]

# 难度分析

- **难度等级**：[简单/较易/中等/较难/困难]
- **原因**：[为什么是这个难度，基于步骤数、知识点数等]

# 常见错误模式

- **错误类型 1**：[描述常见错误，如"混淆公式"]
- **错误类型 2**：[描述常见错误，如"计算失误"]
- **错误类型 3**：[描述常见错误，如"遗漏条件"]

# 建议

**对教师**：
- [教学建议，如何讲解这道题]
- [重点强调哪些知识点]

**对学生**：
- [学习建议，如何掌握这类题目]
- [需要补充哪些基础知识]
```

**要求**：
- 简洁明了，避免冗长
- 基于客观事实，不要过度推测
- 结合知识点分析和难度计算的结果

---

### 步骤 4: 生成解题思路 (thinkingProcess)

使用 `generate_thinking_process_template` 获取模板，然后针对具体题目生成：

**模板**：
```markdown
# 解题思路

## 分析题意

[分析题目给出了什么信息，要求什么]
- 已知条件：[列出已知条件]
- 求解目标：[要求什么]

## 解题方法

[选择什么方法，为什么选择这个方法]
- 方法名称：[如"因式分解法"]
- 选择理由：[为什么用这个方法，而不是其他方法]

## 关键步骤

1. [关键步骤 1，如"建立方程"]
2. [关键步骤 2，如"因式分解"]
3. [关键步骤 3，如"求解未知数"]

## 注意事项

- [注意点 1，如"检查判别式"]
- [注意点 2，如"验证解是否满足定义域"]
```

**要求**：
- 思路清晰，逻辑连贯
- 适合学生阅读理解
- 突出关键思维过程

### 步骤 5: 生成解题步骤 (solutionSteps)

**格式**：
```typescript

  {
    stepNumber: 1,
    title: "理解题意并列出已知条件",
    description: "从题目中提取已知信息，明确求解目标",
    formula: null,
    reasoning: "这一步是为了明确题目给出的条件，避免遗漏",
    commonErrors: [
      "遗漏了某个条件",
      "误解了题目要求"
    ]
  },
  {
    stepNumber: 2,
    title: "设未知数并建立方程",
    description: "设 x 为所求的未知数，根据题意建立方程",
    formula: "ax² + bx + c = 0",
    reasoning: "根据题意建立数学模型，将文字问题转化为数学问题",
    commonErrors: [
      "设错未知数",
      "方程建立错误"
    ]
  },
  {
    stepNumber: 3,
    title: "使用因式分解法求解",
    description: "将方程左边因式分解",
    formula: "(x - x₁)(x - x₂) = 0",
    reasoning: "因式分解可以快速找到方程的根",
    commonErrors: [
      "因式分解错误",
      "忘记检查判别式"
    ]
  },
  {
    stepNumber: 4,
    title: "求解并验证",
    description: "得到 x₁ 和 x₂，验证是否满足原方程",
    formula: "x = x₁ 或 x = x₂",
    reasoning: "验证确保解的正确性",
    commonErrors: [
      "忘记验证",
      "验证步骤错误"
    ]
  }
]
```

**要求**：
- 每个步骤都有明确的标题和描述
- 包含关键公式（如果有）
- 说明推理原因
- 列出常见错误（基于经验和知识点）

### 步骤 6: 生成常见错误 (commonMistakes)

基于题目特点和知识点，预测学生可能犯的错误：

**格式**：
```typescript
[
  {
    description: "忘记检查判别式，导致出现无解的情况",
    frequency: "high",
    knowledgeGaps: ["kp-001", "kp-002"],
    remediation: "加强判别式的学习，理解 Δ = b² - 4ac 的意义。建议做 5-10 道相关练习题。"
  },
  {
    description: "计算错误，符号搞错（如负号遗漏）",
    frequency: "medium",
    knowledgeGaps: ["kp-003"],
    remediation: "注意计算细节，养成检查的习惯。建议逐步计算，避免跳步。"
  },
  {
    description: "混淆完全平方公式和平方差公式",
    frequency: "medium",
    knowledgeGaps: ["kp-005", "kp-006"],
    remediation: "复习公式差异：完全平方公式 (a±b)² = a² ± 2ab + b²，平方差公式 (a+b)(a-b) = a² - b²。"
  }
]
```

**要求**：
- 描述具体，避免模糊表达
- 频率分类：high（>50%学生）、medium（20-50%）、low（<20%）
- 指出受影响的知识点（knowledgeGaps）
- 提供具体的补救措施（remediation）

### 步骤 7: 生成知识缺口分析 (knowledgeGapAnalysis)

分析学生可能存在的知识缺口：

**模板**：
```markdown
# 知识缺口分析

## 必备前置知识

- **[知识点 1]**：[如果缺失，会导致什么问题]
  - 例如："一元一次方程"：如果不掌握，无法理解二次方程的概念
- **[知识点 2]**：[如果缺失，会导致什么问题]
  - 例如："完全平方公式"：如果不掌握，无法进行因式分解

## 常见薄弱环节

1. **[薄弱点 1]**：[描述]
   - 例如："判别式的应用"：很多学生知道公式，但不知道何时使用
2. **[薄弱点 2]**：[描述]
   - 例如："符号处理"：正负号容易搞混

## 补救建议

- [建议 1]：如"先复习一元一次方程，再学习二次方程"
- [建议 2]：如"做 10 道判别式练习题，巩固理解"
- [建议 3]：如"每次计算后检查符号，养成习惯"
```

**要求**：
- 基于知识点依赖关系分析
- 结合常见错误推断薄弱环节
- 提供可操作的补救建议

### 步骤 8: 生成深度难度分析 (difficultyAnalysis)

**不使用数值评分（1-5），而是提供结构化的深度文本分析**

#### 8.1 总体难度概述 (overview)

给出总体难度的文本描述：
- "简单" - 基础概念，步骤少，无陷阱
- "较易" - 常规题型，步骤清晰
- "中等" - 需要综合运用知识点，有一定计算量
- "较难" - 概念抽象或计算复杂，容易出错
- "困难" - 多个知识点综合，步骤多，有陷阱

**示例**：`"中等偏难"`

---

#### 8.2 挑战性方面分析 (challengingAspects)

列出题目的 1-5 个挑战性方面，每个包含：

**格式**：
```typescript
{
  aspect: "概念理解",
  description: "需要深刻理解判别式的几何意义，不仅是公式套用",
  difficulty: "高",
  affectedStudents: "约70%的学生在这里遇到困难"
}
```

**常见挑战方面**：
- 概念理解（抽象概念、定理理解）
- 计算复杂度（多步运算、符号处理）
- 陷阱识别（易忽略的条件、特殊情况）
- 方法选择（多种解法中选择最优）
- 时间压力（计算量大，容易超时）

**要求**：
- 每个方面都有具体描述
- difficulty: "低" | "中" | "高"
- affectedStudents 基于经验估计（如"约50%"）

---

#### 8.3 前置知识要求 (prerequisiteKnowledge)

列出 1-10 个前置知识点，每个包含：

**格式**：
```typescript
{
  topic: "一元一次方程",
  importance: "必需",
  lackImpact: "无法理解题目要求，无从下手"
}
```

**重要性分类**：
- "必需" - 没有这个知识点完全无法解题
- "重要" - 缺失会严重影响解题效率和正确性
- "有帮助" - 有助于更快或更好地理解题目

**要求**：
- 按重要性排序（必需 > 重要 > 有帮助）
- lackImpact 要具体说明影响

---

#### 8.4 常见困难列表 (commonDifficulties)

基于经验预测学生常犯的错误，用文本数组表示：

**示例**：
```typescript
[
  "60%学生混淆完全平方公式和平方差公式",
  "40%学生忘记检查判别式，导致出现增根",
  "30%学生在符号处理上出错，尤其是负号",
  "20%学生跳过验证步骤"
]
```

**要求**：
- 包含估计的错误比例
- 3-8 个常见困难
- 描述具体且可操作

---

#### 8.5 分层时间估算 (timeEstimate)

为不同水平的学生估算时间：

**格式**：
```typescript
{
  fastLearner: "5-8分钟",
  averageLearner: "10-15分钟",
  slowLearner: "20-30分钟",
  rationale: "主要时间花在理解判别式的意义和因式分解上，计算本身不复杂"
}
```

**学生分类**：
- fastLearner: 基础扎实，反应快，很少出错
- averageLearner: 普通学生，需要正常思考时间
- slowLearner: 基础薄弱，需要更多时间理解

**要求**：
- 时间范围合理（fastLearner < averageLearner < slowLearner）
- rationale 说明时间主要花在哪里

---

#### 8.6 适用性分析 (suitableFor)

说明这道题适合什么样的学生：

**格式**：
```typescript
{
  gradeLevel: "初三及以上",
  priorKnowledge: "已学习一元二次方程基本概念，掌握因式分解基础",
  recommendedUse: "作为因式分解单元的巩固练习，或期中复习题"
}
```

**要求**：
- gradeLevel: 建议的年级范围
- priorKnowledge: 需要先掌握什么
- recommendedUse: 推荐的使用场景

### 步骤 9: 生成相关题目推荐 (relatedQuizzes)

使用 `search_quizzes` 查找相似题目：

**调用示例**：
```json
{
  "knowledgePointId": "kp-001",
  "quizType": "解答题",
  "limit": 5
}
```

**⚠️ 重要**：必须过滤掉当前题目本身！

**格式**：
```typescript
[
  {
    id: "quiz-002",
    content: "解方程 2x² - 5x + 2 = 0（用因式分解法）",
    similarity: 0.92,
    similarityReason: "涉及相同的知识点（一元二次方程、因式分解），题型相同（解答题）",
    matchedKnowledgePoints: ["一元二次方程", "因式分解"]
  },
  {
    id: "quiz-003",
    content: "解方程 x² + 6x + 9 = 0",
    similarity: 0.85,
    similarityReason: "涉及相同的知识点（一元二次方程），题型相同，但涉及完全平方公式",
    matchedKnowledgePoints: ["一元二次方程", "完全平方公式"]
  }
]
```

**相似度计算**：
- 知识点重叠度：50%
- 题型相同：50%

**要求**：
- 推荐 3-5 道题目
- 必须过滤掉当前题目
- 说明相似的原因

// 2. 保存知识点标签
await write_output({
  field: "knowledgePointTags",

    { id: "kp-001", name: "一元二次方程", confidence: 0.95, ... },
    { id: "kp-045", name: "判别式", confidence: 0.88, ... }

  preview: "已标注 2 个知识点"

// 3. 保存解题思路
await write_output({

await write_output({
  field: "solutionSteps",

    { stepNumber: 1, title: "...", ... },

  ],
  preview: "已生成 5 个解题步骤"

// 5. 保存常见错误
await write_output({
  field: "commonMistakes",
  value: [

## 性能优化建议

- 相关题目推荐可以缓存（短时间内不变）

### 批量处理
- 如果需要分析多道题目，可以批量处理
- 共享知识点树查询，减少 API 调用
