# 教案优化专家 - 基于崔允漷理论

> **⚠️ 强制要求：在回复用户任何消息之前，你必须首先使用 Read 工具读取 `.context/lesson-plan.json` 文件，了解用户当前正在编辑的备课方案。不要直接询问用户已经在表单中填写的信息。**

## 何时使用

当你需要：
- 优化一份已有的教案
- 检查教案是否符合课程标准要求
- 改进学习目标的表述
- 设计更有效的评价任务
- 将零散的课时设计整合为单元整体设计

## 理论框架

本 Skill 基于华东师范大学崔允漷教授的课程与教学设计理论，核心包括：

### 1. 逆向设计三阶段

```
┌─────────────────────────────────────────────────────────┐
│  阶段一：确定预期结果                                      │
│  · 学生应该知道什么？能做什么？                            │
│  · 哪些是值得理解的大概念？                               │
│  · 期望学生获得哪些核心素养？                              │
├─────────────────────────────────────────────────────────┤
│  阶段二：确定合适的评估证据                                │
│  · 什么证据能证明学生达成了目标？                          │
│  · 设计什么样的表现性任务？                               │
│  · 如何收集学习过程中的证据？                              │
├─────────────────────────────────────────────────────────┤
│  阶段三：设计学习体验和教学                                │
│  · 需要哪些知识和技能？                                   │
│  · 哪些活动能促进理解？                                   │
│  · 如何组织教学顺序？                                     │
└─────────────────────────────────────────────────────────┘
```

### 2. 学习目标的规范表述（ABCD法）

| 要素 | 说明 | 示例 |
|------|------|------|
| A - Audience | 行为主体（学生） | "学生能够..." |
| B - Behavior | 可观察的行为动词 | "分析、比较、解释、创作..." |
| C - Condition | 行为条件 | "在...情境下"、"借助...工具" |
| D - Degree | 达成程度 | "准确地"、"至少三个"、"80%正确率" |

**行为动词分类（布鲁姆认知层次）：**
- 记忆：识别、列举、复述、背诵
- 理解：解释、概括、推断、比较
- 应用：运用、执行、实施、操作
- 分析：区分、组织、归因、解构
- 评价：检验、批判、判断、评估
- 创造：设计、构建、策划、创作

### 3. 大概念提炼

大概念是学科中具有迁移价值的核心观念，特征包括：
- 抽象性：超越具体事实
- 持久性：终身受用的理解
- 可迁移：适用于多种情境
- 整合性：统整多个知识点

**提炼方法：**
1. 从课程标准中的核心概念出发
2. 从学科大问题、大任务出发
3. 从专家思维方式出发
4. 从跨学科主题出发

### 4. 表现性评价任务设计

**GRASPS框架：**
- **G**oal（目标）：任务要达成什么？
- **R**ole（角色）：学生扮演什么角色？
- **A**udience（对象）：成果呈现给谁？
- **S**ituation（情境）：在什么背景下完成？
- **P**roduct（成果）：产出什么作品？
- **S**tandards（标准）：用什么标准评价？

### 5. 素养导向的目标层次

```
┌─────────────────────────────────────────────────────────┐
│                    核心素养目标                          │
│              （学科育人的长期目标）                        │
├─────────────────────────────────────────────────────────┤
│                    单元目标                              │
│          （大概念、大问题、大任务）                        │
├─────────────────────────────────────────────────────────┤
│                    课时目标                              │
│           （具体、可操作、可评价）                         │
└─────────────────────────────────────────────────────────┘
```

## 教案优化流程

### 第一步：诊断原教案

检查以下问题：

**目标层面：**
- [ ] 学习目标是否使用了可观察的行为动词？
- [ ] 目标表述是否符合ABCD规范？
- [ ] 目标是否指向学科核心素养？
- [ ] 目标层次是否合理（不只停留在记忆层面）？

**评价层面：**
- [ ] 评价任务是否与目标匹配？
- [ ] 是否有过程性评价设计？
- [ ] 评价标准是否清晰？
- [ ] 是否设计了表现性任务？

**教学层面：**
- [ ] 教学活动是否指向目标达成？
- [ ] 是否有促进深度理解的活动？
- [ ] 学生主体性是否体现？
- [ ] 时间分配是否合理？

**整体层面：**
- [ ] 是否体现大概念？
- [ ] 是否有真实情境？
- [ ] 是否关注学科思维方式？
- [ ] 是否有迁移应用设计？

### 第二步：重构学习目标

将原目标改写为规范表述：

**改写前（常见问题）：**
```
了解李白的生平
体会诗歌的意境
培养学生的审美能力
```

**改写后（规范表述）：**
```
1. 学生能够借助注释和工具书，准确说出李白的三个重要人生阶段及其代表作品。
   （记忆层次，条件+程度明确）

2. 学生能够运用意象分析法，从至少两个角度解释《静夜思》中"月"的象征意义。
   （理解+分析层次，方法+程度明确）

3. 学生能够仿照《静夜思》的写法，创作一首表达思乡之情的四行小诗，
   至少使用一个具体意象。
   （创造层次，任务+标准明确）
```

### 第三步：设计评价任务

**前置评价（诊断性）：**
- 设计前测问题或任务
- 了解学生起点

**嵌入式评价（形成性）：**
- 课堂观察要点
- 关键问题检测
- 小组讨论评价

**总结性评价（表现性任务）：**
使用GRASPS框架设计：
```
【情境任务示例】
假设你是一位文化推广大使（Role），
需要向外国友人（Audience）介绍中国古典诗词的魅力。
请选择一首唐诗（Situation），
制作一份3分钟的讲解视频或图文海报（Product），
要求：内容准确、有文化深度、能引发兴趣（Standards）。
```

### 第四步：优化教学活动

**活动设计原则：**
1. 目标导向：每个活动都指向特定目标
2. 学生中心：增加学生思考和表达时间
3. 深度学习：设计促进理解的活动
4. 情境真实：创设有意义的学习情境

**活动类型建议：**
| 目标层次 | 适合的活动类型 |
|----------|---------------|
| 记忆 | 识记竞赛、概念配对、填空检测 |
| 理解 | 用自己的话解释、举例说明、概念图 |
| 应用 | 情境问题、案例分析、模拟操作 |
| 分析 | 比较异同、归因探究、结构分析 |
| 评价 | 辩论、作品互评、标准制定 |
| 创造 | 项目设计、作品创作、方案策划 |

**教案模板（崔允漷版）：**

## 多格式输出生成

### 生成教案讲稿

当用户请求"生成讲稿"或"全套材料"时，基于教案内容生成口语化讲稿：

#### 讲稿结构

```markdown
# 教案讲稿：[课题名称]

## 开场白
老师们好，今天我来分享一节关于[主题]的课程设计...

## 课程概述
- 学科：[学科]
- 年级：[年级]
- 课时：[课时]

## 教学目标解读
[用口语化方式解读每个教学目标]

## 教学重难点分析
[分析本课的重点和难点]

## 教学过程讲解
[按环节讲解教学设计思路]

## 评价设计说明
[解释评价任务的设计意图]

## 总结
这节课的设计特点是...
```

### 生成音频 (NotebookLM)

**重要**: 当用户请求"生成音频"或"全套材料"时，调用 `/notebooklm` skill。

#### 语言要求

- 讲稿必须是**简体中文**
- NotebookLM 生成的音频将继承讲稿的语言
- 生成的音频应为中文对话风格

#### 调用方式

```
使用 Skill 工具:
  skill: "notebooklm"
  args: "使用刚才生成的中文教案讲稿创建一个讲解音频。讲稿是简体中文，请生成中文音频。"
```

#### NotebookLM 工作流程

1. **创建 Notebook**
   - 笔记本名称：`教案_{课题名}_{日期}`

2. **上传讲稿作为 Source**
   - 将教案讲稿内容作为文本源添加

3. **生成 Audio Overview**
   - 选择"Audio Overview"功能
   - 预计生成时间：1-3 分钟

4. **下载音频文件**
   - 保存路径：`.agent-workspace/sessions/{sessionId}/outputs/教案讲解音频.mp3`

#### 错误处理

如果 NotebookLM 调用失败：
1. 检查浏览器是否已登录 Google 账号
2. 确认 NotebookLM 服务可用
3. 重试一次
4. 如果仍然失败，告知用户并提供讲稿下载链接

### 生成 PPT

调用 `/pptx` 技能创建教案演示文稿。

**重要**: PPT 内容必须是中文，面向中文用户。

#### 调用方式

```
使用 Skill 工具:
  skill: "example-skills:pptx"
  args: "创建一个中文教案PPT演示文稿。所有内容必须使用简体中文，包括标题、正文、标注等。"
```

#### 幻灯片结构

```
1. 封面 - 课题名称、学科年级（中文）
2. 课程概述 - 基本信息（中文）
3. 教学目标 - 目标列表（中文）
4. 教学重难点 - 重点难点分析（中文）
5. 教学过程 (多页) - 每个环节一页（中文）
6. 评价设计 - 评价任务（中文）
7. 作业设计 - 基础和拓展作业（中文）
8. 板书设计 - 板书示意图（中文）
```

#### 语言要求

- 所有幻灯片标题：简体中文
- 所有正文内容：简体中文
- 图表标注：中文说明
- 提示和备注：中文
