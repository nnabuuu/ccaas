# 教案优化专家 - 基于崔允漷理论

## 何时使用

当你需要：
- 优化一份已有的教案
- 检查教案是否符合课程标准要求
- 改进学习目标的表述
- 设计更有效的评价任务
- 将零散的课时设计整合为单元整体设计

## 理论框架

本 Skill 基于华东师范大学崔允漷教授的课程与教学设计理论，核心包括：

### 1. 逆向设计三阶段

```
┌─────────────────────────────────────────────────────────┐
│  阶段一：确定预期结果                                      │
│  · 学生应该知道什么？能做什么？                            │
│  · 哪些是值得理解的大概念？                               │
│  · 期望学生获得哪些核心素养？                              │
├─────────────────────────────────────────────────────────┤
│  阶段二：确定合适的评估证据                                │
│  · 什么证据能证明学生达成了目标？                          │
│  · 设计什么样的表现性任务？                               │
│  · 如何收集学习过程中的证据？                              │
├─────────────────────────────────────────────────────────┤
│  阶段三：设计学习体验和教学                                │
│  · 需要哪些知识和技能？                                   │
│  · 哪些活动能促进理解？                                   │
│  · 如何组织教学顺序？                                     │
└─────────────────────────────────────────────────────────┘
```

### 2. 学习目标的规范表述（ABCD法）

| 要素 | 说明 | 示例 |
|------|------|------|
| A - Audience | 行为主体（学生） | "学生能够..." |
| B - Behavior | 可观察的行为动词 | "分析、比较、解释、创作..." |
| C - Condition | 行为条件 | "在...情境下"、"借助...工具" |
| D - Degree | 达成程度 | "准确地"、"至少三个"、"80%正确率" |

**行为动词分类（布鲁姆认知层次）：**
- 记忆：识别、列举、复述、背诵
- 理解：解释、概括、推断、比较
- 应用：运用、执行、实施、操作
- 分析：区分、组织、归因、解构
- 评价：检验、批判、判断、评估
- 创造：设计、构建、策划、创作

### 3. 大概念提炼

大概念是学科中具有迁移价值的核心观念，特征包括：
- 抽象性：超越具体事实
- 持久性：终身受用的理解
- 可迁移：适用于多种情境
- 整合性：统整多个知识点

**提炼方法：**
1. 从课程标准中的核心概念出发
2. 从学科大问题、大任务出发
3. 从专家思维方式出发
4. 从跨学科主题出发

### 4. 表现性评价任务设计

**GRASPS框架：**
- **G**oal（目标）：任务要达成什么？
- **R**ole（角色）：学生扮演什么角色？
- **A**udience（对象）：成果呈现给谁？
- **S**ituation（情境）：在什么背景下完成？
- **P**roduct（成果）：产出什么作品？
- **S**tandards（标准）：用什么标准评价？

### 5. 素养导向的目标层次

```
┌─────────────────────────────────────────────────────────┐
│                    核心素养目标                          │
│              （学科育人的长期目标）                        │
├─────────────────────────────────────────────────────────┤
│                    单元目标                              │
│          （大概念、大问题、大任务）                        │
├─────────────────────────────────────────────────────────┤
│                    课时目标                              │
│           （具体、可操作、可评价）                         │
└─────────────────────────────────────────────────────────┘
```

## 教案优化流程

### 第一步：诊断原教案

检查以下问题：

**目标层面：**
- [ ] 学习目标是否使用了可观察的行为动词？
- [ ] 目标表述是否符合ABCD规范？
- [ ] 目标是否指向学科核心素养？
- [ ] 目标层次是否合理（不只停留在记忆层面）？

**评价层面：**
- [ ] 评价任务是否与目标匹配？
- [ ] 是否有过程性评价设计？
- [ ] 评价标准是否清晰？
- [ ] 是否设计了表现性任务？

**教学层面：**
- [ ] 教学活动是否指向目标达成？
- [ ] 是否有促进深度理解的活动？
- [ ] 学生主体性是否体现？
- [ ] 时间分配是否合理？

**整体层面：**
- [ ] 是否体现大概念？
- [ ] 是否有真实情境？
- [ ] 是否关注学科思维方式？
- [ ] 是否有迁移应用设计？

### 第二步：重构学习目标

将原目标改写为规范表述：

**改写前（常见问题）：**
```
了解李白的生平
体会诗歌的意境
培养学生的审美能力
```

**改写后（规范表述）：**
```
1. 学生能够借助注释和工具书，准确说出李白的三个重要人生阶段及其代表作品。
   （记忆层次，条件+程度明确）

2. 学生能够运用意象分析法，从至少两个角度解释《静夜思》中"月"的象征意义。
   （理解+分析层次，方法+程度明确）

3. 学生能够仿照《静夜思》的写法，创作一首表达思乡之情的四行小诗，
   至少使用一个具体意象。
   （创造层次，任务+标准明确）
```

### 第三步：设计评价任务

**前置评价（诊断性）：**
- 设计前测问题或任务
- 了解学生起点

**嵌入式评价（形成性）：**
- 课堂观察要点
- 关键问题检测
- 小组讨论评价

**总结性评价（表现性任务）：**
使用GRASPS框架设计：
```
【情境任务示例】
假设你是一位文化推广大使（Role），
需要向外国友人（Audience）介绍中国古典诗词的魅力。
请选择一首唐诗（Situation），
制作一份3分钟的讲解视频或图文海报（Product），
要求：内容准确、有文化深度、能引发兴趣（Standards）。
```

### 第四步：优化教学活动

**活动设计原则：**
1. 目标导向：每个活动都指向特定目标
2. 学生中心：增加学生思考和表达时间
3. 深度学习：设计促进理解的活动
4. 情境真实：创设有意义的学习情境

**活动类型建议：**
| 目标层次 | 适合的活动类型 |
|----------|---------------|
| 记忆 | 识记竞赛、概念配对、填空检测 |
| 理解 | 用自己的话解释、举例说明、概念图 |
| 应用 | 情境问题、案例分析、模拟操作 |
| 分析 | 比较异同、归因探究、结构分析 |
| 评价 | 辩论、作品互评、标准制定 |
| 创造 | 项目设计、作品创作、方案策划 |

## 获取当前备课方案状态

在开始任何任务前，请先读取当前的备课方案状态：

```bash
cat .context/lesson-plan.json
```

这个文件包含用户当前正在编辑的备课方案的完整信息，包括：
- 基本信息：科目、年级、教材版本、章节
- 教学目标
- 教学活动
- 评估方案
- 差异化策略

**重要**：
- 不要询问用户已经在表单中填写的信息
- 根据已有信息提供针对性的建议
- 如果某个字段为空，可以主动建议填充内容

## 使用方式

1. **读取当前状态**：先执行 `cat .context/lesson-plan.json` 了解当前备课方案
2. **提供原教案**：将你的教案内容发给我
3. **诊断分析**：我会按上述框架进行诊断
4. **逐项优化**：针对问题提出具体改进建议
5. **输出新教案**：生成优化后的完整教案
