# Design Context — KedgeAgentic (即见Agentic)

## Users

**Two distinct audiences, one cohesive experience:**

- **Enterprise developers** (core platform, admin dashboard): Technical users building AI-powered solutions. They need clarity, predictability, and efficient workflows. They interact with the admin dashboard (Refine + shadcn/ui) and APIs.
- **Business end-users** (chat-interface, solutions): Non-technical users — educators, analysts, domain experts — interacting through embedded chat interfaces. They need warmth, approachability, and confidence that the AI understands them.

**Job to be done**: Developers describe business logic in Skills and connect tools via MCP; the platform handles everything else. End-users get natural, reliable AI conversations tailored to their domain.

## Brand Personality

**Warm, Intelligent, Calm**

- **Warm**: Terracotta accent (`#AE5630`), off-white backgrounds (`#F5F5F0`), serif typography for AI responses — never clinical or cold.
- **Intelligent**: Typography-led design, clear information hierarchy, precise spacing — the interface feels considered, not decorated.
- **Calm**: Generous whitespace, subtle shadows, smooth animations (`ease-claude`, `ease-claude-spring`) — no visual noise or urgency.

## Aesthetic Direction

**Reference**: claude.ai — minimal, typography-focused, warm neutrals.

**Visual tone**: Quiet confidence. The interface should feel like a well-designed book — readable, inviting, and unhurried.

**Key visual rules**:
- Warm neutrals only — no pure `#000`, `#FFF`, or cold grays
- Sans-serif for user input and UI chrome; serif (Georgia) for AI responses
- User messages: right-aligned bubbles with warm taupe background
- Assistant messages: no bubble, bare serif text with generous line height (`1.65rem`)
- Composer: floating card with `rounded-[20px]` and shadow, no visible border
- Terracotta `#AE5630` accent for interactive elements (send button, links on hover)
- `font-medium` (500) for emphasis — never `font-bold` (700)
- Dark mode mirrors warmth: `#262624` background, `#FAF9F5` text — not inverted cold

**Theme**: Light and dark mode, both warm.

## Design Principles

1. **Typography is the interface** — Let type do the heavy lifting. Spacing, weight, and font choice communicate hierarchy more effectively than color or decoration. Serif for AI, sans for humans.

2. **Warm over neutral** — Every surface, border, and shadow should feel warm. Cold grays and pure black/white break the experience. Use `#F5F5F0` (light) and `#262624` (dark) as base.

3. **Quiet confidence** — No flashy animations, gradient backgrounds, or attention-grabbing UI. Subtle shadows, smooth transitions (`ease-claude`), and `active:scale-[0.98]` feedback. The product should feel calm and reliable.

4. **Accessibility is non-negotiable** — WCAG 2.1 AA compliance. Ensure contrast ratios meet standards, keyboard navigation works, and screen readers can parse content. Support `prefers-reduced-motion` and `prefers-color-scheme`.

5. **Two audiences, one language** — Developer-facing surfaces (admin) can be denser and more utilitarian (shadcn/ui patterns). User-facing surfaces (chat) must be spacious, warm, and approachable. Both share the same color DNA and typographic voice.

## Typography

| Role | Font | Weight | Line Height |
|------|------|--------|-------------|
| UI / User messages | System sans-serif | 400 | 1.4 |
| AI responses | Georgia, serif | 400 | 1.65rem |
| Code | ui-monospace | 400 | relaxed |
| Emphasis | (inherit) | 500 (medium) | (inherit) |

## Animation

| Curve | Value | Use |
|-------|-------|-----|
| ease-claude | `cubic-bezier(0.4, 0, 0.2, 1)` | General transitions |
| ease-claude-spring | `cubic-bezier(0.165, 0.85, 0.45, 1)` | Interactive feedback |
| Button press | `active:scale-[0.98]` | Standard buttons |
| Send button press | `active:scale-95` | Primary action |
